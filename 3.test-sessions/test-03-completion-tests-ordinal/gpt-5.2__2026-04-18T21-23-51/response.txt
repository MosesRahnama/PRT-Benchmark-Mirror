Below are **Lean proof skeletons** you can drop into the three remaining `sorry` cases. They’re structured to make the ordinal reasoning “modular”: isolate a **big dominating principal ordinal** of the form `omega0 ^ X`, prove each summand is `< omega0 ^ X`, then close by principal-closure-under-addition and monotonicity.

I’m intentionally leaving a few internal sub-lemmas as “fill here” spots—because the exact lemma names in your Mathlib version can vary—but the *shape* of the proof (and the ordinal facts you need) should be stable.

## 1) `R_eq_refl`: positivity of `omega0 ^ (...) + 1`

Goal after `simp [mu]` is essentially `0 < omega0 ^ (mu a + mu a + 9) + 1`.

```lean
| _, _, Step.R_eq_refl a => by
    simp [mu]
    -- goal: 0 < omega0 ^ (mu a + mu a + (9 : Ordinal)) + 1
    -- It suffices to show 0 < 1 ≤ (ω^E + 1).
    have h0 : (0 : Ordinal) < (1 : Ordinal) := one_pos
    have h1 : (1 : Ordinal) ≤ omega0 ^ (mu a + mu a + (9 : Ordinal)) + 1 := by
      -- `le_add_left` gives: 1 ≤ X + 1 for any X
      simpa [add_comm, add_left_comm, add_assoc] using
        (le_add_left (1 : Ordinal) (omega0 ^ (mu a + mu a + (9 : Ordinal))))
    exact lt_of_lt_of_le h0 h1
```

Notes:
- This avoids needing any `opow_pos` lemma at all; it’s just `0 < 1` and `1 ≤ X + 1`.

## 2) `R_rec_succ`: show `mu (app ...) < mu (recDelta ... (delta n))`

After `simp [mu]`, the goal is (up to definitional unfolding) of the form:
\[
\underbrace{\omega^3(\mu s+1) + \omega^2(\mu(\mathrm{rec}\;n)+1) + 1}_{\mu(\mathrm{app}\;s\;(\mathrm{rec}\;n))}
\;<\;
\underbrace{\omega^{X} + \omega(\mu b+1)+1}_{\mu(\mathrm{rec}\;(\delta n))}
\]
where \(X := \mu(\delta n) + \mu s + 6\).

The key is: **prove LHS < ω^X**; then it’s automatically `< ω^X + ω*(...) + 1`.

## 3) `R_eq_diff`: show `mu (integrate (merge a b)) < mu (eqW a b)`

After `simp [mu]`, the goal is essentially:
\[
\omega^4(\mu(\mathrm{merge}\ a\ b)+1)+1 \;<\; \omega^{E}+1
\]
where \(E := \mu a + \mu b + 9\).

A clean skeleton is: prove `omega0^(4) * (mu (merge a b) + 1) < omega0^E`, then add `+1` on both sides.

```lean
| _, _, Step.R_eq_diff a b => by
    simp [mu]
    set E : Ordinal := mu a + mu b + (9 : Ordinal)

    -- Reduce to bounding the pre-+1 part
    have hMul : omega0 ^ (4 : Ordinal) * (mu (merge a b) + 1) < omega0 ^ E := by
      -- It suffices to show: mu(merge a b) + 1 < omega0 ^ (mu a + mu b + 5),
      -- then multiply on the left by ω^4 and rewrite exponents:
      --   ω^4 * ω^(mu a + mu b + 5) = ω^(4 + (mu a + mu b + 5)) = ω^E.
      --
      -- Skeleton:

      have h_merge : mu (merge a b) + 1 < omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
        -- Expand mu(merge a b):
        --   mu(merge a b) = ω^3*(μa+1) + ω^2*(μb+1) + 1
        -- So mu(merge a b) + 1 = ω^3*(μa+1) + ω^2*(μb+1) + 2.
        --
        -- Show each summand is < ω^(μa+μb+5) and combine via principality of ω^(...).
        --
        -- You’ll likely want:
        --   hPrin : IsPrincipal (omega0 ^ (mu a + mu b + 5))
        -- plus monotonicity of `omega0 ^ _` and absorption of finite left-mults.
        sorry

      -- Multiply the bound by ω^4 on the left
      have := (mul_lt_mul_left' (omega0 ^ (4 : Ordinal)) h_merge)
      -- Now rewrite:
      --   ω^4 * ω^(μa+μb+5) = ω^(4 + (μa+μb+5)) = ω^(μa+μb+9) = ω^E
      -- using `opow_add` and ordinal arithmetic simp.
      -- Finish with simp/assoc/comm normalization.
      simpa [E, opow_add, add_assoc, add_left_comm, add_comm] using this

    -- Add 1 to both sides
    have h : omega0 ^ (4 : Ordinal) * (mu (merge a b) + 1) + 1 < omega0 ^ E + 1 :=
      add_lt_add_right hMul 1

    -- Match the exact goal shape
    simpa [E, add_assoc, add_left_comm, add_comm] using h
```

What you’ll fill in:
- The lemma `mul_lt_mul_left'` exists for monotone left-multiplication on ordinals in many Mathlib versions; if not, you can rewrite as `lt_of_lt_of_le` + `mul_le_mul_left'` with strictness handled separately.
- `opow_add` rewriting: `omega0 ^ (A + B) = omega0 ^ A * omega0 ^ B`. Sometimes it’s `opow_add` or `opow_add` with arguments reversed; adjust accordingly.
