### `R_rec_succ`

This is the main technical case. The clean way is:

1. show the exponent in the recursive-counter side strictly increases when replacing `n` by `delta n`;
2. lift that strict inequality through `omega0 ^ (...)`;
3. show the whole `app`-measure is absorbed by that much larger leading `omega`-power;
4. then finish by adding the positive tail `+ omega0 * (mu b + 1) + 1`.

```lean
    -- ω^3 * (mu s + 1)
```

### `R_eq_diff`

This case has the same overall shape as `R_rec_succ`, but simpler: the RHS is a single enormous `omega`-power, and the LHS is only `omega^4 * (...) + 1`.

A good proof pattern is:

1. bound the `merge` body by a single `omega`-power of lower rank;
2. multiply by `omega^4`;
3. compare with `omega^(mu a + mu b + 9)`.

Skeleton:

```lean
| _, _, Step.R_eq_diff a b => by
    simp [mu]
    -- Goal:
    -- ω^4 * (ω^3 * (mu a + 1) + ω^2 * (mu b + 1) + 1 + 1) + 1
    --   < ω^(mu a + mu b + 9) + 1

    -- Step 1: compress the `merge` body under one ω-power.
    have hMergeBody :
        omega0 ^ (3 : Ordinal) * (mu a + 1) +
            omega0 ^ (2 : Ordinal) * (mu b + 1) + 1 + 1
          < omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
      -- Idea:
      --   the body is a CNF sum with top exponents 3 and 2,
      --   so it is below a single `ω^(mu a + mu b + 5)`.
      --
      -- In a polished proof, this is another good helper lemma.
      sorry

    -- Step 2: multiplying by ω^4 keeps the whole left side below ω^(mu a + mu b + 9).
    have hIntCore :
        omega0 ^ (4 : Ordinal) *
            (omega0 ^ (3 : Ordinal) * (mu a + 1) +
              omega0 ^ (2 : Ordinal) * (mu b + 1) + 1 + 1)
          < omega0 ^ (mu a + mu b + (9 : Ordinal)) := by
      -- Use `hMergeBody`, monotonicity of multiplication, and
      -- the ordinal law `ω^4 * ω^γ = ω^(4 + γ)` / comparison with that form.
      --
      -- Since `γ = mu a + mu b + 5`, the target exponent becomes `... + 9`.

    -- Step 3: add `+ 1` on both sides.

        omega0 ^ (4 : Ordinal) *
            (omega0 ^ (3 : Ordinal) * (mu a + 1) +

          < omega0 ^ (mu a + mu b + (9 : Ordinal)) + 1 := by
```

## A cleaner way to finish the hard two

If this were my next pass, I would not try to brute-force `R_rec_succ` and `R_eq_diff` inline. I’d first prove one or two helper lemmas of the form:

```lean
lemma small_cnf_lt_big_opow
  {α β : Ordinal} :
  omega0 ^ (3 : Ordinal) * (α + 1) + omega0 ^ (2 : Ordinal) * (β + 1) + 1
    < omega0 ^ (α + β + (5 : Ordinal)) := by
  sorry
```

and possibly

```lean
lemma app_measure_lt_rec_succ_head
  (b s n : Trace) :
  mu (app s (recDelta b s n))
    < omega0 ^ (mu (delta n) + mu s + (6 : Ordinal)) := by
  sorry
```

Then:

- `R_eq_diff` becomes one helper invocation plus `+ 1`;
- `R_rec_succ` becomes one exponent-growth lemma plus one helper invocation.

That usually makes Lean much less grumpy. Ordinals are majestic, but they do enjoy paperwork.
