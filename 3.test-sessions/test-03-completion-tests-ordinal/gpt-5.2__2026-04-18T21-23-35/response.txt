## Case 1: `Step.R_rec_succ` skeleton

Goal: for `Step (recDelta b s (delta n)) (app s (recDelta b s n))`, prove
`mu (app s (recDelta b s n)) < mu (recDelta b s (delta n))`.

Key idea: the **leading term** of `mu (recDelta ... (delta n))` is

- `omega0 ^ (mu (delta n) + mu s + 6)`

and this exponent contains `mu (delta n) = omega0^5 * (mu n + 1) + 1`, which is enormous. Show the entire `mu (app ...)` is `< omega0 ^ (mu (delta n) + mu s + 6)`, then finish by “adding a positive tail preserves `<`”.

```lean
| _, _, Step.R_rec_succ b s n => by
  -- Unfold `mu` on both sides
  simp [mu]

  -- After `simp`, your goal should look like:
  -- ω^3*(mu s + 1) + ω^2*(mu (recDelta b s n) + 1) + 1
  --   < ω^(mu (delta n) + mu s + 6) + ω*(mu b + 1) + 1

  -- Let E be the big exponent on the RHS leading term.
  set E : Ordinal := mu (delta n) + mu s + (6 : Ordinal)

  -- It suffices to show:
  --   mu(app ...) < ω^E
  -- because then `mu(app ...) < ω^E ≤ ω^E + ω*(...) + 1`.
  have h_tail : omega0 ^ E ≤ omega0 ^ E + omega0 * (mu b + 1) + 1 := by
    -- two `le_add_right` steps
    exact le_trans (le_add_right _ _) (le_add_right _ _)

  -- (ii) use additive principal property of ω^E (since `0 < E`) to combine.

  have hEpos : (0 : Ordinal) < E := by
    -- `E = mu (delta n) + mu s + 6`, and `6 > 0`.
    -- show `0 < 6` then `0 < E` by `lt_of_lt_of_le` with `le_add_left` etc

  have hPowE_pos : (0 : Ordinal) < omega0 ^ E := opow_pos' E

  -- First summand: ω^3*(mu s + 1) < ω^E

    -- show `(3 : Ordinal) < E`, then `ω^3 ≤ ω^E`, then multiply by `(mu s + 1)` carefully
    -- or show directly `ω^3*(mu s+1) < ω^(3+something) ≤ ω^E`
    -- ...
    sorry

  -- Second summand: ω^2*(mu(recDelta ...) + 1) < ω^E
  have h2 : omega0 ^ (2 : Ordinal) * (mu (recDelta b s n) + 1) < omega0 ^ E := by

      -- unfold `mu (delta n)` and show `mu n < ω^5*(mu n + 1) + 1`
      -- mimic the style from your merge proofs:

      --   ...

      -- you will still need an inequality; keep it as a sub-proof.

      -- add monotonicity: from `hn_lt` get
      --   mu n + mu s + 6 < mu (delta n) + mu s + 6 = E
      -- via `add_lt_add_right` twice (or `lt_of_lt_of_le` with `le_add_left`)

    -- and propagate through `ω^2 * _`.
    -- ...

    -- ...
```

## Case 2: `Step.R_eq_refl` skeleton (easy positivity)

Here you just need `0 < omega0 ^ (...) + 1`.

```lean
| _, _, Step.R_eq_refl a => by
  simp [mu]
  -- goal: 0 < omega0 ^ (mu a + mu a + 9) + 1
  -- immediate from `+ 1` on the right:
  exact add_pos_right _ one_pos
```

If `add_pos_right` doesn’t fire due to typeclass expectations, a fallback is:

```lean
  have : (0 : Ordinal) < (1 : Ordinal) := one_pos
  exact lt_of_lt_of_le this (le_add_left _ _)
```

(But `add_pos_right _ one_pos` is the cleanest.)

## Case 3: `Step.R_eq_diff` skeleton

Goal: `mu (integrate (merge a b)) < mu (eqW a b)`.

After `simp [mu]` you’ll be at something like:

- LHS: `omega0^4 * (mu (merge a b) + 1) + 1`
- RHS: `omega0^(mu a + mu b + 9) + 1`

A standard pattern:

1. Prove the stronger inequality  
   `omega0^4 * (mu (merge a b) + 1) < omega0^(mu a + mu b + 9)`.
2. Then apply `add_lt_add_right` (add `+1` to both sides).

The hard part is step (1): bound `mu (merge a b) + 1` by some `omega0^(mu a + mu b + 5)` so that multiplying by `omega0^4` lands below `omega0^(mu a + mu b + 9)` via `opow_add`.

Skeleton:

```lean
| _, _, Step.R_eq_diff a b => by
  simp [mu]
  -- goal:
  -- ω^4 * (μ(merge a b) + 1) + 1 < ω^(μ a + μ b + 9) + 1

  -- It suffices to prove the part before `+1`, then add `+1` to both sides.
  apply (add_lt_add_right_iff).1?  -- if convenient; otherwise use `have` + `exact add_lt_add_right`
  -- More robust:
  have hcore :
    omega0 ^ (4 : Ordinal) * (mu (merge a b) + 1) < omega0 ^ (mu a + mu b + (9 : Ordinal)) := by

    -- Step A: show `mu (merge a b) + 1 < omega0 ^ (mu a + mu b + 5)`
    have hmerge :
      mu (merge a b) + 1 < omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
      -- Expand `mu (merge a b)`:
      --   μ(merge a b) = ω^3*(μ a + 1) + ω^2*(μ b + 1) + 1
      simp [mu]
      -- goal becomes: ω^3*(μ a + 1) + ω^2*(μ b + 1) + 1 + 1 < ω^(μ a + μ b + 5)
      -- Show each summand < ω^(μ a + μ b + 5), then combine using additive principalness.
      -- Key subgoals:
      --   ω^3*(μ a + 1) < ω^(μ a + μ b + 5)
      --   ω^2*(μ b + 1) < ω^(μ a + μ b + 5)
      --   2 < ω^(μ a + μ b + 5)
      -- and `0 < μ a + μ b + 5`.
      sorry

    -- Step B: rewrite RHS exponent as `(4 + (mu a + mu b + 5))` and use `opow_add`
    -- so that: ω^4 * ω^(μa+μb+5) = ω^(4 + (μa+μb+5)) = ω^(μa+μb+9).
    have hopow_add :
      omega0 ^ (mu a + mu b + (9 : Ordinal))
        = omega0 ^ (4 : Ordinal) * omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
      -- `opow_add` typically gives `ω^(x+y) = ω^x * ω^y`
      -- Use associativity/commutativity of `+` to massage `μa+μb+9` into `4 + (μa+μb+5)`.
      -- ...
      sorry

    -- Step C: finish via monotonicity of left multiplication by `ω^4`.
    -- From `hmerge`, get `ω^4*(μ(merge)+1) < ω^4*ω^(μa+μb+5) = ω^(μa+μb+9)`.
    -- ...
    sorry

  -- Now lift `hcore` to the original goal by adding 1:
  exact add_lt_add_right hcore 1
```

The essential “missing lemma work” is all inside `hmerge` (bounding the merge expression by a single $\omega$-power) plus the `opow_add` rearrangement. Once you have that, the rest is purely bookkeeping.
