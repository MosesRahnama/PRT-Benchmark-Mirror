### `R_rec_succ`

This is the real boss fight. Also, your comment block currently labels the two sides backwards: after `simp [mu]`, the **left** side is `mu (app s (recDelta b s n))`, and the **right** side is `mu (recDelta b s (delta n))`.

A good skeleton is:

```lean
| _, _, Step.R_rec_succ b s n => by
    simp [mu]
    have hdom :
        omega0 ^ (3 : Ordinal) * (mu s + 1) +
            omega0 ^ (2 : Ordinal) *
              (omega0 ^ (mu n + mu s + (6 : Ordinal)) +
                omega0 * (mu b + 1) + 1 + 1) + 1
          <
        omega0 ^
          (omega0 ^ (5 : Ordinal) * (mu n + 1) + 1 + mu s + (6 : Ordinal)) := by
      /-
      Main domination lemma.

      Heuristic:
      - the lhs is built from finitely many terms with top growth coming from
          ω^2 * ω^(μ n + μ s + 6)
        together with lower-order polynomial-in-ω pieces;
      - the rhs is a single principal term
          ω^(ω^5 * (μ n + 1) + 1 + μ s + 6),
        whose exponent contains the huge contribution
          ω^5 * (μ n + 1),
        dwarfing anything appearing on the lhs.

      In practice, prove this by:
      1. bounding the lhs by some intermediate ordinal;
      2. showing that intermediate ordinal is below the rhs by exponent comparison.
      -/
      sorry

    calc
      omega0 ^ (3 : Ordinal) * (mu s + 1) +
          omega0 ^ (2 : Ordinal) *
            (omega0 ^ (mu n + mu s + (6 : Ordinal)) +
              omega0 * (mu b + 1) + 1 + 1) + 1
        <
          omega0 ^
            (omega0 ^ (5 : Ordinal) * (mu n + 1) + 1 + mu s + (6 : Ordinal)) := hdom
      _ ≤
          omega0 ^
            (omega0 ^ (5 : Ordinal) * (mu n + 1) + 1 + mu s + (6 : Ordinal)) +
            omega0 * (mu b + 1) + 1 := by
            exact le_add_right _ _
```

If you want a slightly more modular version, break `hdom` into two internal lemmas:

```lean
have hbound_lhs :

  /-
  or some other convenient intermediate bound;
  choose whatever exponent is easiest to establish.
  -/
  sorry

have hexp :
    mu n + mu s + (8 : Ordinal)
      <
    omega0 ^ (5 : Ordinal) * (mu n + 1) + 1 + mu s + (6 : Ordinal) := by
  /-
  Here is where you use the fact that
    ω^5 * (μ n + 1)
  massively dominates the linear expression
    μ n + μ s + 8.
  -/
  sorry
```

## What to isolate as helper lemmas

If you want the completion to be less brittle, the best move is to pull the hard ordinal arithmetic into named lemmas just above `mu_decreases`. For example:

```lean
lemma eq_diff_bound (a b : Trace) :
    omega0 ^ (4 : Ordinal) *
        (omega0 ^ (3 : Ordinal) * (mu a + 1) +
          omega0 ^ (2 : Ordinal) * (mu b + 1) + 1 + 1)
      <
    omega0 ^ (mu a + mu b + (9 : Ordinal)) := by
  sorry

lemma rec_succ_bound (b s n : Trace) :
    omega0 ^ (3 : Ordinal) * (mu s + 1) +
        omega0 ^ (2 : Ordinal) *
          (omega0 ^ (mu n + mu s + (6 : Ordinal)) +
            omega0 * (mu b + 1) + 1 + 1) + 1
      <
    omega0 ^
      (omega0 ^ (5 : Ordinal) * (mu n + 1) + 1 + mu s + (6 : Ordinal)) := by
  sorry
```

Then the two theorem cases become pleasantly short:

```lean
| _, _, Step.R_eq_diff a b => by
    simp [mu]
    exact add_lt_add_right (eq_diff_bound a b) 1
```

```lean
| _, _, Step.R_rec_succ b s n => by
    simp [mu]
    calc
      _ < omega0 ^
            (omega0 ^ (5 : Ordinal) * (mu n + 1) + 1 + mu s + (6 : Ordinal)) :=
          rec_succ_bound b s n
      _ ≤ _ := by
          exact le_add_right _ _
```

That keeps the main proof readable and quarantines the gnarly ordinal algebra where it belongs.
