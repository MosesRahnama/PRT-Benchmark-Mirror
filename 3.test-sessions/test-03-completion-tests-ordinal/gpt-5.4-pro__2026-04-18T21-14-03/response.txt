Two of the three gaps admit a standard skeleton immediately; `R_rec_succ` is the only one that appears to need a real auxiliary domination lemma, and possibly a stronger measure if that lemma does not go through.

### 1. `R_eq_refl`

This one is routine positivity.

```lean
-- eqW a a --> void
| _, _, Step.R_eq_refl a => by
    simp [mu]
    exact add_pos_right _ one_pos
```

If `add_pos_right` does not close it directly, the slightly more explicit version is:

```lean
| _, _, Step.R_eq_refl a => by
    simp [mu]
    exact Ordinal.pos_iff_ne_zero.mpr (by
      apply ne_of_gt
      exact add_pos_right _ one_pos)
```

### 2. `R_eq_diff`

A clean way is to isolate the inner `merge` size, bound it by a single $\omega$-power, then multiply once more by `ω^4`.

```lean
-- eqW a b --> integrate (merge a b)
| _, _, Step.R_eq_diff a b => by
    simp [mu]

    let α : Ordinal := mu a + mu b + (9 : Ordinal)
    let β : Ordinal :=
      omega0 ^ (3 : Ordinal) * (mu a + 1) +
      omega0 ^ (2 : Ordinal) * (mu b + 1) + 2

    have hβ : β < omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
      have ha :
          omega0 ^ (3 : Ordinal) * (mu a + 1)
            < omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
        -- Bound the coefficient by an ω-power:
        --   mu a + 1 ≤ ω^(mu a + 1)
        -- hence
        --   ω^3 * (mu a + 1) ≤ ω^3 * ω^(mu a + 1) = ω^(3 + (mu a + 1))
        -- and compare exponents:
        --   3 + (mu a + 1) < mu a + mu b + 5
        sorry

      have hb :
          omega0 ^ (2 : Ordinal) * (mu b + 1)
            < omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
        -- Same pattern as `ha`
        sorry

      have h2 : (2 : Ordinal) < omega0 ^ (mu a + mu b + (5 : Ordinal)) := by
        -- Any positive ω-power dominates finite ordinals
        sorry

      -- Combine `ha`, `hb`, `h2`; absorb the finite tail `+ 2`
      sorry

    have hmul :
        omega0 ^ (4 : Ordinal) * β < omega0 ^ α := by
      -- Turn `hβ` into a bound after left-multiplication by `ω^4`.
      -- Conceptually:
      --   β < ω^(mu a + mu b + 5)
      -- so
      --   ω^4 * β < ω^4 * ω^(mu a + mu b + 5) = ω^(4 + (mu a + mu b + 5))
      -- and `4 + (mu a + mu b + 5) < mu a + mu b + 9`.
      sorry

    simpa [α, β, add_assoc, add_left_comm, add_comm] using
      add_lt_add_right hmul 1
```

### 3. `R_rec_succ`

First, the comment in the file has the source/target measures reversed. The goal is:

$$
\mu(\mathrm{app}\ s\ (\mathrm{recDelta}\ b\ s\ n))
<
\mu(\mathrm{recDelta}\ b\ s\ (\delta n)).
$$

A usable skeleton is:

```lean
-- recDelta b s (delta n) --> app s (recDelta b s n)
| _, _, Step.R_rec_succ b s n => by
    simp [mu]

    let α : Ordinal := mu (delta n) + mu s + (6 : Ordinal)

    have hs :
        omega0 ^ (3 : Ordinal) * (mu s + 1) < omega0 ^ α := by
      -- Bound `mu s + 1` by an ω-power, then compare exponents:
      --   ω^3 * (mu s + 1) ≤ ω^(3 + (mu s + 1)) < ω^α
      -- because `mu (delta n)` contributes a huge leading term.
      sorry

    have hrec :
        mu (recDelta b s n) + 1 < omega0 ^ (mu (delta n) + mu s + (4 : Ordinal)) := by
      -- This is the real missing lemma.
      -- It must show the recursive-call measure stays far below the tower
      -- generated by `mu (delta n)`.
      --
      -- Expand:
      --   mu (recDelta b s n)
      --     = ω^(mu n + mu s + 6) + ω * (mu b + 1) + 1
      --
      -- Then prove the whole expression is still < ω^(mu (delta n) + mu s + 4).
      --
      -- If this subgoal resists proof, the present `mu` is probably too weak:
      -- the exponent of `recDelta` may need to depend on `mu b` as well.
      sorry

    have hr :
        omega0 ^ (2 : Ordinal) * (mu (recDelta b s n) + 1) < omega0 ^ α := by
      -- From `hrec`:
      --   ω^2 * (mu (recDelta b s n) + 1)
      --     < ω^2 * ω^(mu (delta n) + mu s + 4)
      --     = ω^(2 + (mu (delta n) + mu s + 4))
      --     < ω^α
      sorry

    have happ :
        omega0 ^ (3 : Ordinal) * (mu s + 1) +
          omega0 ^ (2 : Ordinal) * (mu (recDelta b s n) + 1)
          < omega0 ^ α := by
      -- Combine `hs` and `hr`, then absorb the lower summand.
      sorry

    simpa [α, add_assoc, add_left_comm, add_comm] using
      add_lt_add_right happ 1
```

### Important note on `R_rec_succ`

This is the only place where there may be a genuine arithmetic gap in the current measure. The proof needs a lemma of the form

```lean
mu (recDelta b s n) + 1 < omega0 ^ (mu (delta n) + mu s + 4)
```

but `mu (recDelta b s n)` contains the tail `ω * (mu b + 1)`, and the top exponent for `recDelta` does **not** depend on `mu b`. If that lemma cannot be established, the likely fix is to strengthen the `recDelta` measure so its top exponent also sees `mu b`.

If wanted, a next step is to turn the `R_eq_diff` skeleton into a more concrete sequence of local ordinal lemmas.
