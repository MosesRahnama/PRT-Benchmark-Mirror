## Soundness Assessment of the Lexicographic Termination Measure

The proposed measure is `measure(t) = (phase(t), cost(t))` ordered lexicographically on $\mathbb{N} \times \mathbb{N}$. For soundness, every rule `Step lhs rhs` must satisfy `measure(lhs) > measure(rhs)` in the lexicographic order â€” meaning either `phase` strictly decreases, or `phase` stays equal and `cost` strictly decreases.

Let's check each rule:

### Rules where `phase = 0 â†’ 0` (second component must strictly decrease)

| Rule | LHS | RHS | `cost(lhs)` | `cost(rhs)` | OK? |
|---|---|---|---|---|---|
| **R_int_delta** | `integrate (delta t)` | `void` | $2 + c(t)$ | $0$ | âœ“ |
| **R_merge_void_left** | `merge void t` | `t` | $1 + c(t)$ | $c(t)$ | âœ“ |
| **R_merge_void_right** | `merge t void` | `t` | $1 + c(t)$ | $c(t)$ | âœ“ |
| **R_merge_cancel** | `merge t t` | `t` | $1 + 2c(t)$ | $c(t)$ | âœ“ |
| **R_rec_zero** | `recDelta b s void` | `b` | $1 + c(b) + c(s)$ | $c(b)$ | âœ“ |
| **R_eq_refl** | `eqW a a` | `void` | $3 + 2c(a)$ | $0$ | âœ“ |

### The problematic rules

**R_rec_succ** â€” `recDelta b s (delta n)  âŸ¶  app s (recDelta b s n)`

- `phase(lhs) = 1` (matches `.recDelta _ _ (.delta _)`), `phase(rhs) = 0` (an `app` node). Phase **strictly decreases**, so this is fine on the first component.

**However**, this only accounts for a *single* step. The RHS `app s (recDelta b s n)` contains the subterm `recDelta b s n`. If the rewrite system allows reduction under context (inside `app`), then `recDelta b s n` may itself need to be reduced. If `n` is again of the form `delta n'`, then `recDelta b s n` has `phase = 1` â€” but `app s (recDelta b s n)` as a whole has `phase = 0`. The measure only works if you're measuring the **root redex**, not subterms.

**R_eq_diff** â€” `eqW a b  âŸ¶  integrate (merge a b)`

- `phase(lhs) = 0`, `phase(rhs) = 0`. So `cost` must strictly decrease.
- `cost(lhs) = 3 + c(a) + c(b)`
- `cost(rhs) = 1 + 1 + c(a) + c(b) = 2 + c(a) + c(b)`
- $3 + c(a) + c(b) > 2 + c(a) + c(b)$ âœ“

### Verdict: **The measure is sound for single root-step reduction, but has a critical gap for contextual rewriting.**

The file defines `Step` as a relation on whole terms (not under context â€” there are no congruence/compatibility rules). Under this exact formalization, **every rule strictly decreases the measure**, so the measure is sound for proving that `Step` is well-founded (no infinite chain of root rewrites).

**However**, there is a latent design concern: `R_rec_succ` produces `app s (recDelta b s n)`, which embeds a copy of `s` in the result. If `Step` were ever extended with congruence closure (reducing inside `app`, `merge`, etc.), the `cost` of `s` is duplicated in the RHS. Specifically:

- `cost(lhs) = 1 + c(b) + c(s) + 1 + c(n)` = $2 + c(b) + c(s) + c(n)$
- `cost(rhs) = 1 + c(s) + 1 + c(b) + c(s) + c(n)$ = $2 + 2c(s) + c(b) + c(n)$

So `cost(rhs) - cost(lhs) = c(s)`, meaning **cost increases** on this rule. The phase drop from 1 to 0 saves it for the single step, but under a compatible closure the measure would not be well-founded because recursive unfolding duplicates `s` unboundedly.

### Summary

1. **Sound as written**: For the bare `Step` relation (root rewriting only, no congruence rules), the lexicographic measure $(phase, cost)$ is strictly decreasing on every rule. The relation is well-founded.

2. **Not extensible to contextual rewriting**: `R_rec_succ` duplicates `s` in the RHS, causing `cost` to increase. The `phase` drop compensates for exactly one step, but under repeated reduction in context, this breaks. A multiset path ordering or a measure accounting for recursion depth would be needed.

3. **Minor note**: `R_merge_cancel` collapses `merge t t â†’ t`, which is only valid when the two children are syntactically identical. This is correctly captured by the Lean pattern `âˆ€ t, Step (merge t t) t`, but semantically it assumes a notion of equality on traces that may need justification in a broader formalization.
