## Analysis and Proof Skeletons

### Case 1: R_int_delta (`integrate (delta t) → void`)

**Measure analysis:**
- LHS: `kappa = kappa(t)`, `mu = 1 + mu(t)`  
- RHS: `kappa = 0`, `mu = 0`

**Strategy:** Case split on whether `kappa(t) = 0`
- If `kappa(t) > 0`: kappa drops, use `drop_left`
- If `kappa(t) = 0`: kappa ties, mu drops from `1 + mu(t)` to `0`, use `drop_right`

```lean
| _, _, Step.R_int_delta t => by
    by_cases h : kappa t = 0
    · -- kappa ties at 0, mu drops
      have hmu : mu void < mu (integrate (delta t)) := by simp [mu]
      have hk  : kappa void = kappa (integrate (delta t)) := by simp [kappa, h]
      exact drop_right hmu hk
    · -- kappa drops
      have hk : kappa void < kappa (integrate (delta t)) := by
        simp [kappa]; omega
      exact drop_left hk
```

### Case 2: R_rec_zero (`recDelta b s void → b`)

**Measure analysis:**
- LHS: `kappa = max(kappa(b), kappa(s))`, `mu = 3 + mu(b) + mu(s)`
- RHS: `kappa = kappa(b)`, `mu = mu(b)`

**Strategy:** Case split on whether `kappa(b) = max(kappa(b), kappa(s))`
- If `kappa(b) < max(...)`: kappa drops (when `kappa(s) > kappa(b)`), use `drop_left`
- If equal: kappa ties, mu drops by at least 3, use `drop_right`

```lean
| _, _, Step.R_rec_zero b s => by
    by_cases h : kappa b = Nat.max (kappa b) (kappa s)
    · -- kappa ties, mu drops
      have hmu : mu b < mu (recDelta b s void) := by simp [mu]; omega
      have hk  : kappa b = kappa (recDelta b s void) := by
        simp [kappa]; exact h
      exact drop_right hmu hk
    · -- kappa drops
      have hk : kappa b < kappa (recDelta b s void) := by
        simp [kappa]; omega
      exact drop_left hk
```

### Case 3: R_rec_succ (`recDelta b s (delta n) → app s (recDelta b s n)`)

**Measure analysis:**
- LHS: `kappa = max(max(kappa(b), kappa(s)), kappa(n)) + 1`
- RHS: `kappa = max(kappa(s), max(max(kappa(b), kappa(s)), kappa(n))) = max(max(kappa(b), kappa(s)), kappa(n))`

**Strategy:** Kappa *always* drops by exactly 1 (the recDelta-under-delta nesting unwinds). Use `drop_left`.

```lean
| _, _, Step.R_rec_succ b s n => by
    -- kappa drops by 1 (unwinding recDelta-under-delta)
    have hk : kappa (app s (recDelta b s n)) < kappa (recDelta b s (delta n)) := by
      simp [kappa]; omega
    exact drop_left hk
```

**Strategy:** Same pattern as R_int_delta - case split on `kappa(a)`.

| _, _, Step.R_eq_refl a => by

### Case 5: R_eq_diff (`eqW a b → integrate (merge a b)`)

**Measure analysis:**
- LHS: `kappa = max(kappa(a), kappa(b))`, `mu = 4 + mu(a) + mu(b)`
- RHS: `kappa = max(kappa(a), kappa(b))` (delta/integrate transparent), `mu = 3 + mu(a) + mu(b)`

**Strategy:** Kappa *always* ties, mu drops by exactly 1. Use `drop_right`.

```lean
| _, _, Step.R_eq_diff a b => by
    -- kappa ties, mu drops by 1
    have hmu : mu (integrate (merge a b)) < mu (eqW a b) := by simp [mu]; omega
    have hk  : kappa (integrate (merge a b)) = kappa (eqW a b) := by simp [kappa]
    exact drop_right hmu hk
```

## Summary

- **R_rec_succ** and **R_eq_diff** are the simplest: kappa drops unconditionally, or kappa ties unconditionally respectively
- **R_int_delta**, **R_eq_refl**, and **R_rec_zero** require case analysis on kappa equality
- The key insight is that delta-transparency in both measures, combined with the +1 increment for `recDelta _ _ (delta _)`, creates the strict decrease needed for termination
