## Per-case analysis

### Measure recap

| Term | `kappa` | `mu` |
|---|---|---|
| `void` | 0 | 0 |
| `delta t` | `kappa t` | `mu t` |
| `integrate t` | `kappa t` | `1 + mu t` |

| `eqW a b` | `max(κa, κb)` | `4 + μa + μb` |

### Case 1 — `R_int_delta t` : `integrate (delta t) → void`

**Measures:**
- LHS: `(kappa t, 1 + mu t)` — since delta and integrate are both kappa/mu-transparent
- RHS: `(0, 0)`

**Strategy:** Either `kappa t > 0` (first component drops → `Prod.Lex.left`) or `kappa t = 0` and `0 < 1 + mu t` (second component drops → `Prod.Lex.right`). Both branches close by `omega`.

```lean
| _, _, Step.R_int_delta t => by
    unfold LexOrder measure
    simp [kappa, mu]
    by_cases hk : kappa t = 0
    · rw [hk]; exact Prod.Lex.right _ (by omega)
    · exact Prod.Lex.left _ _ (by omega)
```

---

### Case 2 — `R_rec_zero b s` : `recDelta b s void → b`

**Measures:**
- LHS: `(max(kappa b, kappa s), 3 + mu b + mu s)` — third arg is `void` so the general `recDelta` pattern applies, `kappa void = 0`, `mu void = 0`
- RHS: `(kappa b, mu b)`

**Strategy:** Either `kappa b < max(kappa b, kappa s)` (first component drops) or `kappa b = max(kappa b, kappa s)` and `mu b < 3 + mu b + mu s` (trivially true). Needs `simp [kappa]` to reduce `kappa (recDelta b s void)` since there is no dedicated simp lemma for the non-delta third-argument case.

```lean
| _, _, Step.R_rec_zero b s => by
    unfold LexOrder measure
    simp [kappa, mu]
    -- Goal: Prod.Lex .. (kappa b, mu b) (max (kappa b) (kappa s), 3 + mu b + mu s)
    by_cases hk : kappa b < Nat.max (kappa b) (kappa s)
    · exact Prod.Lex.left _ _ hk
    · have hk_eq : kappa b = Nat.max (kappa b) (kappa s) := by omega
      rw [← hk_eq]
      exact Prod.Lex.right _ (by omega)
```

### Case 3 — `R_rec_succ b s n` : `recDelta b s (delta n) → app s (recDelta b s n)`

**Measures:**
- LHS: `kappa = max(max(κb, κs), κn) + 1`, `mu = 3 + μb + μs + μn`
- RHS: `kappa = max(κs, kappa(recDelta b s n))`, `mu = 1 + μs + (3 + μb + μs + μn) = 4 + μb + 2·μs + μn`

**Sub-case analysis on `n`:**

| `n` shape | `kappa(recDelta b s n)` | RHS kappa vs LHS kappa | mu comparison |
|---|---|---|---|
| `void`, `integrate _`, `merge _ _`, `app _ _`, `recDelta _ _ _`, `eqW _ _` | `max(max(κb, κs), κn)` | Drops by 1 ✓ | N/A (kappa already wins) |
| `delta m` | `max(max(κb, κs), κm) + 1 = LHS kappa` | **Ties** | RHS mu = LHS mu + 1 + μs — **increases** ✗ |

**Diagnosis:** When `n = delta m`, both kappa ties and mu *increases*, so the current `(kappa, mu)` measure is **insufficient** for this sub-case. The six non-delta sub-cases close by `drop_left` after `cases n`.

The issue is that `kappa` detects *at most one* delta layer on the third argument of `recDelta`. When `n` is itself `delta m`, the stripped `recDelta b s (delta m)` still triggers the `+1`, so kappa doesn't decrease.

**Partial skeleton (closes 6 of 7 sub-cases):**

```lean
| _, _, Step.R_rec_succ b s n => by
    unfold LexOrder measure
    cases n with
    | void =>
        simp [kappa, mu]
        exact Prod.Lex.left _ _ (by omega)
    | integrate t =>
        simp [kappa, mu]
        exact Prod.Lex.left _ _ (by omega)
    | merge a b' =>
        simp [kappa, mu]
        exact Prod.Lex.left _ _ (by omega)
    | app a b' =>
        simp [kappa, mu]
        exact Prod.Lex.left _ _ (by omega)
    | recDelta b' s' n' =>
        simp [kappa, mu]
        exact Prod.Lex.left _ _ (by omega)
    | eqW a b' =>
        simp [kappa, mu]
        exact Prod.Lex.left _ _ (by omega)
    | delta m =>
        -- ⚠ GAP: kappa ties, mu increases. Current measure is insufficient.
        -- FIX: replace `kappa` with a delta-depth–aware variant, e.g.
        --
        --   def deltaDepth : Trace → Nat
        --     | .delta t => 1 + deltaDepth t
        --     | _        => 0
        --
        --   def kappa : Trace → Nat
        --     | .recDelta b s n => max(max(kappa b, kappa s), kappa n) + deltaDepth n
        --     | ...             -- other clauses unchanged
        --
        -- With this definition:
        --   LHS kappa = max(max(κb,κs), κm) + 1 + deltaDepth m
        --   RHS kappa(recDelta b s (delta m)) = max(max(κb,κs), κm) + deltaDepth(delta m)
        --                                     = max(max(κb,κs), κm) + 1 + deltaDepth m
        --   ... no, still ties. The correct fix:
        --
        --   | .recDelta b s n => max(max(kappa b, kappa s), kappa n) + deltaDepth n + 1
        --   ... and adjust the non-delta case to also use this.
        --
        -- Alternatively, use a triple measure (kappa, deltaDepth-of-3rd-arg, mu)
        -- where the middle component counts nested deltas in the recDelta's 3rd arg.
        sorry
```

**Recommended fix** (requires modifying the file, noted here for completeness): introduce a middle component `deltaDepth` to the lexicographic tuple, making the measure a triple `(kappa, deltaArg, mu)` where:

```lean
def deltaArg : Trace → Nat
  | .recDelta _ _ n => deltaLayers n
  | _               => 0

def deltaLayers : Trace → Nat
  | .delta t => 1 + deltaLayers t
  | _        => 0
```

Under `R_rec_succ` with `n = delta m`: kappa ties, `deltaArg` drops by 1 (one delta layer stripped), closing the case regardless of mu.

### Case 4 — `R_eq_refl a` : `eqW a a → void`

**Measures:**
- LHS: `(kappa a, 4 + 2·mu a)` — since `max(κa, κa) = κa` and `4 + μa + μa`
- RHS: `(0, 0)`

**Strategy:** Identical structure to `R_int_delta` — case split on `kappa a`.

```lean
| _, _, Step.R_eq_refl a => by
    unfold LexOrder measure
    simp [kappa, mu]
    by_cases hk : kappa a = 0
    · rw [hk]; exact Prod.Lex.right _ (by omega)
    · exact Prod.Lex.left _ _ (by omega)
```

### Case 5 — `R_eq_diff a b` : `eqW a b → integrate (merge a b)`

**Measures:**
- LHS: `(max(κa, κb), 4 + μa + μb)`
- RHS: `(max(κa, κb), 3 + μa + μb)` — since `kappa (integrate (merge a b)) = max(κa, κb)` and `mu(integrate(merge a b)) = 1 + 2 + μa + μb`

**Strategy:** Kappa ties exactly; mu drops by 1. Pure `drop_right`.

```lean
| _, _, Step.R_eq_diff a b => by
    have hmu : mu (integrate (merge a b)) < mu (eqW a b) := by simp [mu]; omega
    have hk  : kappa (integrate (merge a b)) = kappa (eqW a b) := by simp [kappa]
    exact drop_right hmu hk
```

## Summary

| Case | Strategy | Status |
|---|---|---|
| `R_int_delta` | case split on `kappa t`; both branches close | **closes** |
| `R_rec_zero` | case split on `kappa b` vs `max(κb,κs)`; both branches close | **closes** |
| `R_rec_succ` | cases on `n`: 6 non-delta constructors close via `drop_left`; `n = delta m` **fails** — measure increases | **gap at `delta m`** |
| `R_eq_refl` | case split on `kappa a`; both branches close | **closes** |
| `R_eq_diff` | `drop_right` directly; kappa ties, mu drops by 1 | **closes** |

Four of five remaining cases close under the current measure. The `R_rec_succ`/`delta m` sub-case is a genuine gap: the `(kappa, mu)` pair is too coarse to detect the delta-layer being stripped. The minimal fix is to promote the measure to a **triple** `(kappa, deltaLayers(3rd-arg-of-recDelta), mu)` so that the middle component drops by 1 whenever one delta wrapper is peeled off the recursive argument.
