Here is the analysis and proof skeleton for all five remaining cases.

## Four provable cases

### 1. `R_int_delta` — `integrate (delta t) → void`

| Component | LHS `integrate (delta t)` | RHS `void` |
|-----------|--------------------------|------------|
| `kappa`   | `kappa t`                | `0`        |
| `mu`      | `1 + mu t`               | `0`        |

If `kappa t > 0`, the first lex component drops strictly. If `kappa t = 0`, kappa ties and `mu` drops (`0 < 1 + mu t`).

```lean
| _, _, Step.R_int_delta t => by
    by_cases hk : kappa t = 0
    · -- kappa ties at 0, mu drops: 0 < 1 + mu t
      exact drop_right (by simp [mu]; omega) (by simp [kappa]; omega)
    · -- kappa drops strictly: 0 < kappa t
      exact drop_left (by simp [kappa]; omega)
```

### 2. `R_rec_zero` — `recDelta b s void → b`

| Component | LHS `recDelta b s void`          | RHS `b`   |
|-----------|----------------------------------|-----------|
| `kappa`   | `max (max (kappa b) (kappa s)) 0` = `max (kappa b) (kappa s)` | `kappa b` |
| `mu`      | `3 + mu b + mu s`                | `mu b`    |

`kappa b ≤ max (kappa b) (kappa s)`. If `kappa s > kappa b`, the first component drops strictly. Otherwise they tie, and `mu b < 3 + mu b + mu s`.

```lean
| _, _, Step.R_rec_zero b s => by
    by_cases hk : kappa b = kappa (recDelta b s void)
    · -- kappa ties, mu drops: mu b < 3 + mu b + mu s
      exact drop_right (by simp [mu]; omega) hk
    · -- kappa drops: kappa b < max (kappa b) (kappa s)
      exact drop_left (by simp [kappa] at hk ⊢; omega)
```

### 3. `R_eq_refl` — `eqW a a → void`

| Component | LHS `eqW a a`              | RHS `void` |
|-----------|----------------------------|------------|
| `kappa`   | `max (kappa a) (kappa a)` = `kappa a` | `0`  |
| `mu`      | `4 + 2 * mu a`             | `0`        |

Same pattern as `R_int_delta`: case-split on whether `kappa a = 0`.

```lean
| _, _, Step.R_eq_refl a => by
    by_cases hk : kappa a = 0
    · -- kappa ties at 0, mu drops: 0 < 4 + 2 * mu a
      exact drop_right (by simp [mu]; omega) (by simp [kappa]; omega)
    · -- kappa drops: 0 < kappa a
      exact drop_left (by simp [kappa]; omega)
```

### 4. `R_eq_diff` — `eqW a b → integrate (merge a b)`

| Component | LHS `eqW a b`              | RHS `integrate (merge a b)` |
|-----------|----------------------------|-----------------------------|
| `kappa`   | `max (kappa a) (kappa b)`  | `max (kappa a) (kappa b)`   |
| `mu`      | `4 + mu a + mu b`          | `1 + 2 + mu a + mu b` = `3 + mu a + mu b` |

Kappa is identical on both sides; `mu` drops by exactly 1. No case split needed.

```lean
| _, _, Step.R_eq_diff a b => by
    have hmu : mu (integrate (merge a b)) < mu (eqW a b) := by simp [mu]; omega
    have hk  : kappa (integrate (merge a b)) = kappa (eqW a b) := by simp [kappa]
    exact drop_right hmu hk
```

## One unprovable case

### `R_rec_succ` — `recDelta b s (delta n) → app s (recDelta b s n)`

This case **cannot be closed** with the current `(kappa, mu)` measure. Here is a concrete counterexample:

Set `b = void`, `s = void`, `n = delta void`:

| | term | `kappa` | `mu` |
|---|---|---|---|
| LHS | `recDelta void void (delta (delta void))` | `max(max(0,0), 0) + 1 = 1` | `3 + 0 + 0 + 0 = 3` |
| RHS | `app void (recDelta void void (delta void))` | `max(0, max(max(0,0),0)+1) = 1` | `1 + 0 + (3+0+0+0) = 4` |

The measure moves from `(1, 3)` to `(1, 4)` — it **increases** in the second component while the first component ties. The lexicographic order is violated.

**Root cause.** `kappa` awards `+1` only for one layer of `delta`-wrapping in the third argument of `recDelta`. Peeling off a `delta` removes that `+1`, but the remaining `recDelta b s n` term can itself have `n` delta-headed, re-gaining the `+1`. Meanwhile `mu` strictly increases because the RHS introduces an extra `app s (…)` wrapper (adding `1 + mu s`).

**Suggested fix.** Replace the single `+1` bonus in `kappa` with a count of all leading `delta` layers in the third argument:

```lean
def deltaDepth : Trace → Nat
  | .delta t => 1 + deltaDepth t
  | _        => 0

-- In the kappa definition, replace the two recDelta clauses with:
| .recDelta b s n => Nat.max (Nat.max (kappa b) (kappa s)) (kappa n) + deltaDepth n
```

With this change, `deltaDepth (delta n) = 1 + deltaDepth n`, so the `kappa` component of the LHS is always strictly greater than that of the RHS, regardless of `n`'s shape. The proof then closes via `drop_left` for all `n`.
