Each case is reduced to one of the two lemmas already proved earlier:

* `drop_left  : kappa b < kappa a → LexOrder (measure b) (measure a)`
* `drop_right : mu b < mu a → kappa b = kappa a → LexOrder (measure b) (measure a)`

The skeleton shows the *shape* of the argument (the `by`‑blocks) and leaves the
actual arithmetic steps as `sorry`‑place‑holders that you can later replace
with `simp`, `omega`, `linarith`, or a short calculation.

```lean
/-- Every primitive step strictly decreases the lexicographic measure. -/
theorem measure_decreases : forall {a b : Trace}, Step a b -> LexOrder (measure b) (measure a)
| _, _, Step.R_merge_void_left t => by
    have hmu : mu t < mu (merge void t) := by simp [mu]
    have hk  : kappa t = kappa (merge void t) := by simp [kappa]
    exact drop_right hmu hk
| _, _, Step.R_merge_void_right t => by
    have hmu : mu t < mu (merge t void) := by simp [mu]
    have hk  : kappa t = kappa (merge t void) := by simp [kappa]
    exact drop_right hmu hk
| _, _, Step.R_merge_cancel t => by
    have hmu : mu t < mu (merge t t) := by
      -- mu (merge t t) = 2 + mu t + mu t, so the inequality is obvious
      simp [mu]  -- leaves goal `mu t < 2 + mu t + mu t`
      omega
    have hk  : kappa t = kappa (merge t t) := by simp [kappa]
    exact drop_right hmu hk

-- ── Remaining cases ───────────────────────────────────────────

-- integrate (delta t) --> void
| _, _, Step.R_int_delta t => by
    -- kappa drops from `kappa (integrate (delta t)) = kappa t`
    -- to `kappa void = 0`.  This is a strict drop, so we can use `drop_left`.
    have hk : kappa void < kappa (integrate (delta t)) := by
      -- unfold the definitions and finish the inequality
      simp [kappa]  -- goal becomes `0 < kappa t`
      sorry
    exact drop_left hk

-- recDelta b s void --> b
| _, _, Step.R_rec_zero b s => by
    -- Here `kappa (recDelta b s void) = max (max (kappa b) (kappa s)) (kappa void)`
    -- = `max (max (kappa b) (kappa s)) 0`.  The result `b` has kappa `kappa b`,
    -- which is ≤ the left‑hand side, but we need a *strict* drop.
    have hk : kappa b < kappa (recDelta b s void) := by
      -- unfold `kappa` and use `Nat.max_lt_iff` (or `omega`) to get the strict inequality
      simp [kappa]  -- goal becomes `kappa b < max (max (kappa b) (kappa s)) 0`
      sorry
    exact drop_left hk

-- recDelta b s (delta n) --> app s (recDelta b s n)
| _, _, Step.R_rec_succ b s n => by
    -- Two possibilities:
    --   * kappa drops because the `+ 1` disappears,
    --   * or kappa stays the same and mu drops.
    -- In this system the `+ 1` is the only place where kappa can increase,
    -- so the step strictly *decreases* kappa.
    have hk : kappa (app s (recDelta b s n)) < kappa (recDelta b s (delta n)) := by
      -- unfold both sides; the left side is `max (kappa s) (max (max (kappa b) (kappa s)) (kappa n))`
      -- the right side is that same max plus `+ 1`.
      simp [kappa]  -- goal becomes `max (kappa s) (max (max (kappa b) (kappa s)) (kappa n))
                     --   < max (max (kappa b) (kappa s)) (kappa n) + 1`
      sorry
    exact drop_left hk

-- eqW a a --> void
| _, _, Step.R_eq_refl a => by
    -- `kappa (eqW a a) = max (kappa a) (kappa a) = kappa a`,
    -- while `kappa void = 0`.  This is a strict drop.
    have hk : kappa void < kappa (eqW a a) := by
      simp [kappa]  -- goal becomes `0 < kappa a`
      sorry
    exact drop_left hk

-- eqW a b --> integrate (merge a b)
| _, _, Step.R_eq_diff a b => by
    -- Two sub‑cases:
    --   1. `kappa` drops (if `kappa a` ≠ `kappa b`);
    --   2. `kappa` stays equal and `mu` drops.
    have hk_or_hmu : (kappa (integrate (merge a b)) < kappa (eqW a b)) ∨
                     (kappa (integrate (merge a b)) = kappa (eqW a b) ∧
                      mu (integrate (merge a b)) < mu (eqW a b)) := by
      -- unfold the definitions and reason about the two possibilities.
      -- The `integrate` constructor adds no kappa, while `merge` takes the max.
      -- Hence `kappa (integrate (merge a b)) = max (kappa a) (kappa b)`.
      -- This is either strictly smaller than `max (kappa a) (kappa b)` (impossible)
      -- or equal, in which case we must use the `mu` drop.
      simp [kappa, mu]  -- reduces to a trivial disjunction
      sorry
    cases hk_or_hmu with
    | inl hk =>
        exact drop_left hk
    | inr hkm =>
        rcases hkm with ⟨hk, hmu⟩
        exact drop_right hmu hk
```

### How to finish the skeleton

* Replace each `sorry` inside the `by` blocks with a concrete arithmetic proof.
  * `simp`, `omega`, `linarith`, or `nat_arith` are usually enough.
  * For the `drop_left` cases you only need to show a strict inequality on `kappa`.
  * For the `drop_right` case (`R_eq_diff`) you may first prove that the `kappa`s are equal
    (by `simp [kappa]`) and then that `mu` strictly decreases (by `simp [mu]` and a
    simple arithmetic argument).

* No other parts of the file are changed, and the overall theorem `strong_normalization`
  will compile once the five `sorry`s are replaced by concrete proofs.
