### Case 1 â `R_int_delta` â `integrate (delta t) âĶ void`

`kappa` falls from `kappa t` to `0`. This is a direct `drop_left` unless `kappa t = 0`, in which case `kappa` ties and `mu` falls from `1 + mu t` to `0`.

```lean
-- integrate (delta t) --> void
-- kappa(LHS) = kappa t   kappa(RHS) = 0
-- mu(LHS)    = 1 + mu t  mu(RHS)    = 0
| _, _, Step.R_int_delta t => by
    by_cases hk : kappa t = 0
    Â· -- kappa ties at 0; mu drops: 0 < 1 + mu t
      have hmu : mu void < mu (integrate (delta t)) := by simp [mu_integrate, mu_delta, mu_void]; omega
      have hkeq : kappa void = kappa (integrate (delta t)) := by
        simp [kappa_integrate, kappa_delta, kappa_void, hk]
      exact drop_right hmu hkeq
    Â· -- kappa strictly drops: 0 < kappa t
      have hklt : kappa void < kappa (integrate (delta t)) := by
        simp [kappa_integrate, kappa_delta, kappa_void]
        omega          -- uses hk : kappa t â  0
      exact drop_left hklt
```

### Case 2 â `R_rec_zero` â `recDelta b s void âĶ b`

### Case 3 â `R_rec_succ` â `recDelta b s (delta n) âĶ app s (recDelta b s n)` â ïļ

`kappa(LHS) = K + 1` where `K = max (max Îšb Îšs) Îšn` (by `kappa_rec_delta`). After `simp [kappa_app, kappa_rec_delta]` the goal becomes:

$$\max(\kappa_s,\; \texttt{kappa(recDelta b s n)}) < K + 1$$

This requires an auxiliary bound `kappa (recDelta b s n) âĪ K`. A `cases n` exposes the problem: for all non-`delta` constructors the second branch of `kappa` applies and the bound holds; `omega` closes those sub-goals. **For the `delta n'` sub-case, however, `kappa (recDelta b s (delta n')) = K + 1` (equal to `kappa(LHS)`) so `drop_left` fails, and `mu(RHS) = 4 + Îžb + 2Îžs + Îžn' > 3 + Îžb + Îžs + Îžn' = mu(LHS)` so `drop_right` also fails. The current `(kappa, mu)` measure is insufficient for this sub-case.**

```lean

-- kappa(LHS)  = max(max Îšb Îšs)(Îšn) + 1   [kappa_rec_delta]
-- kappa(RHS)  = max Îšs (kappa(recDelta b s n))
-- mu(LHS)     = 3 + Îžb + Îžs + Îžn
-- mu(RHS)     = 4 + Îžb + 2Â·Îžs + Îžn   â strictly LARGER for all n
| _, _, Step.R_rec_succ b s n => by
    apply drop_left
    -- Goal: kappa (app s (recDelta b s n)) < kappa (recDelta b s (delta n))
    simp only [kappa_app, kappa_rec_delta]
    -- Goal: max (kappa s) (kappa (recDelta b s n)) < max (max (kappa b) (kappa s)) (kappa n) + 1
    -- Need: kappa (recDelta b s n) âĪ max (max Îšb Îšs) (Îšn)
    -- Proceed by cases on n to expose which kappa branch applies.
    cases n with
    | void =>
        -- kappa (recDelta b s void) = max (max Îšb Îšs) 0 = max Îšb Îšs âĪ K
        show Nat.max (kappa s)
               (Nat.max (Nat.max (kappa b) (kappa s)) (kappa void))
             < Nat.max (Nat.max (kappa b) (kappa s)) (kappa void) + 1
        simp [kappa_void]; omega
    | delta n' =>
        -- kappa (recDelta b s (delta n')) = max (max Îšb Îšs) Îšn' + 1 = K + 1
        -- kappa (LHS) = K + 1 where K = max (max Îšb Îšs) (kappa (delta n')) = max (max Îšb Îšs) Îšn'
        -- drop_left fails: kappa(RHS) = K+1 = kappa(LHS)
        -- drop_right also fails: mu strictly increases
        -- *** The (kappa, mu) measure is not a decreasing measure for this sub-case.
        -- A fix requires either a deeper kappa (counting all delta-wrapped recDeltas
        -- in the whole term), or a third lexicographic component (e.g. structural size
        -- not transparent to delta).
        sorry
    | integrate t =>
        show Nat.max (kappa s)
               (Nat.max (Nat.max (kappa b) (kappa s)) (kappa (integrate t)))
             < Nat.max (Nat.max (kappa b) (kappa s)) (kappa (integrate t)) + 1
        simp [kappa_integrate]; omega
    | merge a c =>
        show Nat.max (kappa s)
               (Nat.max (Nat.max (kappa b) (kappa s)) (kappa (merge a c)))
             < Nat.max (Nat.max (kappa b) (kappa s)) (kappa (merge a c)) + 1
        simp [kappa_merge]; omega
    | app f x =>
        show Nat.max (kappa s)
               (Nat.max (Nat.max (kappa b) (kappa s)) (kappa (app f x)))
             < Nat.max (Nat.max (kappa b) (kappa s)) (kappa (app f x)) + 1
        simp [kappa_app]; omega
    | recDelta b' s' n' =>
        -- third arg is recDelta b' s' n' (not delta-wrapped at top level)
        -- kappa(recDelta b s (recDelta b' s' n')) uses the non-delta branch
        show Nat.max (kappa s)
               (Nat.max (Nat.max (kappa b) (kappa s)) (kappa (recDelta b' s' n')))
             < Nat.max (Nat.max (kappa b) (kappa s)) (kappa (recDelta b' s' n')) + 1
        omega
    | eqW x y =>
        show Nat.max (kappa s)
               (Nat.max (Nat.max (kappa b) (kappa s)) (kappa (eqW x y)))
             < Nat.max (Nat.max (kappa b) (kappa s)) (kappa (eqW x y)) + 1
        simp [kappa_eqW]; omega
```

## Summary

| Case | Strategy | Obstacle |
|---|---|---|
| `R_int_delta` | `by_cases` on `kappa t = 0`; `drop_right` / `drop_left` | none |
| `R_rec_zero` | `by_cases` on `kappa s âĪ kappa b`; unfold `kappa` at `void` arg; `drop_right` / `drop_left` | need `show` to expose the non-delta kappa branch |
| `R_rec_succ` | `drop_left` + `cases n`; seven non-delta constructors close with `omega` | the `delta n'` sub-case is unprovable with `(kappa, mu)`: both kappa ties **and** mu increases |
| `R_eq_refl` | `by_cases` on `kappa a = 0`; `drop_right` / `drop_left` | none |
| `R_eq_diff` | direct `drop_right` | none |

**Diagnostic note on `R_rec_succ` / `delta n'` sub-case:** when `n = delta n'`, the step is `recDelta b s (delta (delta n')) âĶ app s (recDelta b s (delta n'))`. Both sides have `kappa = max(max Îšb Îšs)(Îšn') + 1`, so kappa does not decrease; and `mu(RHS) â mu(LHS) = 1 + mu s âĨ 1`, so mu strictly *increases*. The lexicographic `(kappa, mu)` order is not a valid termination certificate for this rule with the current definitions. A fix would be to redefine `kappa` to sum delta-wrapped `recDelta` occurrences across the *entire* term (not just add +1 at the immediate node), so that the outer +1 is not replicated in the reduct.
