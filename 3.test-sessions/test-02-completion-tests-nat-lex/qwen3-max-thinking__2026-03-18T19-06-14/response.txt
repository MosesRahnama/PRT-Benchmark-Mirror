2. **Max Function Properties**: Use properties of `Nat.max` to handle comparisons, particularly that `max(x, y) ≥ x` and `max(x, y) ≥ y`.

3. **Mu Monotonicity**: Leverage that mu is always positive for non-void terms and that constructor weights ensure strict decreases.

4. **Lexicographic Ordering**: Apply `drop_left` when kappa strictly decreases, and `drop_right` when kappa ties but mu strictly decreases.
