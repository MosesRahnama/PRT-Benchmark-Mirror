## `R_int_delta` : `integrate (delta t) → void`

You need a **case split on `kappa t`**: if it’s positive, you get a left-drop; if it’s `0`, you use a right-drop via `mu`.

```lean
| _, _, Step.R_int_delta t => by
    -- Goal: LexOrder (measure void) (measure (integrate (delta t))).
    cases h : kappa t with
    | zero =>
        have hk : kappa void = kappa (integrate (delta t)) := by
          -- both are 0 in this branch
          simp [kappa, h]
        have hmu : mu void < mu (integrate (delta t)) := by
          -- 0 < 1 + mu t
          simp [mu]
        exact drop_right hmu hk
    | succ k =>
        have hk : kappa void < kappa (integrate (delta t)) := by
          -- 0 < succ k
          simpa [kappa, h] using (Nat.succ_pos k)
        exact drop_left hk
```

## `R_rec_zero` : `recDelta b s void → b`

Here `kappa` may or may not strictly drop, so do a **`≤` then split into `<`/`=`**. If equal, `mu` drops strictly because `recDelta` adds a positive constant and extra structure.

```lean
| _, _, Step.R_rec_zero b s => by
    have hmu : mu b < mu (recDelta b s void) := by
      -- mu(recDelta b s void) = 3 + mu b + mu s
      simp [mu]
      -- Finish with any Nat arithmetic tactic/lemma you prefer:
      -- e.g. omega / nlinarith / exact Nat.lt_of_lt_of_le ...
      omega

    have hk_le : kappa b ≤ kappa (recDelta b s void) := by
      -- kappa(recDelta b s void) = max (max (kappa b) (kappa s)) 0
      simp [kappa]
      -- kb ≤ max(kb,ks) ≤ max(max(kb,ks),0)
      exact le_trans (Nat.le_max_left _ _) (Nat.le_max_left _ _)

    cases lt_or_eq_of_le hk_le with
    | inl hklt =>
        exact drop_left hklt
    | inr hkeq =>
        exact drop_right hmu hkeq
```

## `R_rec_succ` : `recDelta b s (delta n) → app s (recDelta b s n)`

### The intended skeleton (and the snag)

You’d *like* to prove a strict `kappa`-drop (then use `drop_left`), because `mu` **actually increases** in this step:
\[
\mu(\text{app } s (\text{recDelta } b\, s\, n)) = 4 + \mu b + 2\mu s + \mu n
\]
\[
\mu(\text{recDelta } b\, s\, (\delta n)) = 3 + \mu b + \mu s + \mu n
\]
so $\mu$ goes up by $1+\mu s$.

So the proof must be `drop_left`, i.e. show
`kappa (app s (recDelta b s n)) < kappa (recDelta b s (delta n))`.

A natural attempt is to **split on `n`** so that `kappa (recDelta b s n)` simplifies (because its definition pattern-matches on whether `n` is `delta _`):

```lean
| _, _, Step.R_rec_succ b s n => by
    -- Goal: LexOrder (measure (app s (recDelta b s n)))
    --              (measure (recDelta b s (delta n))).
    -- Since mu increases, we must use drop_left (kappa strictly drops).
    -- The definition of kappa for (recDelta b s n) does a case split on n,
    -- so do it explicitly here.
    cases n with
    | void =>
        -- "good" branch: kappa strictly drops by the outer +1 on the LHS
        have hk : kappa (app s (recDelta b s void)) < kappa (recDelta b s (delta void)) := by
          -- After simp, this should reduce to something like M < M + 1
          -- with M := max(max kb ks) 0, plus a max-equality using M ≥ ks.
          simp [kappa]
          -- Typically:
          --   1) set M := Nat.max (Nat.max (kappa b) (kappa s)) 0
          --   2) show Nat.max (kappa s) M = M (since kappa s ≤ M)
          --   3) conclude M < M+1 via Nat.lt_succ_self
          admit
        exact drop_left hk

    | delta n' =>
        -- ⚠️ Problem branch with the current definition:
        --   LHS: recDelta b s (delta (delta n')) has kappa = M + 1 where M := max(max kb ks) (kappa n')
        --   RHS: app s (recDelta b s (delta n')) has kappa = max(ks, M+1) = M+1
        -- so kappa TIES (not <), while mu increases, so neither drop_left nor drop_right applies.
        --
        -- Conclusion: this case cannot be closed with the current (kappa, mu).
        --
        -- To finish, you typically refine the measure, e.g.:
        --   * make kappa count the length of the delta-chain in the 3rd argument (a true "recursion height"),
        --   * or use a 3-component lex measure where the middle component decreases here,
        --   * or (since Step is root-only) use a measure that ignores subterms under app.
        --
        -- Skeleton placeholder once a refined measure is in place:
        -- have hk : kappa_refined (app s (recDelta b s (delta n'))) <
        --           kappa_refined (recDelta b s (delta (delta n'))) := by ...
        -- exact drop_left hk
        sorry

    | integrate n' =>
        have hk : kappa (app s (recDelta b s (integrate n'))) < kappa (recDelta b s (delta (integrate n'))) := by
          simp [kappa]
          admit
        exact drop_left hk

    | merge a b' =>
        have hk : kappa (app s (recDelta b s (merge a b'))) < kappa (recDelta b s (delta (merge a b'))) := by
          simp [kappa]
          admit
        exact drop_left hk

    | app a b' =>
        have hk : kappa (app s (recDelta b s (app a b'))) < kappa (recDelta b s (delta (app a b'))) := by
          simp [kappa]
          admit
        exact drop_left hk

    | recDelta b' s' n' =>
        have hk : kappa (app s (recDelta b s (recDelta b' s' n'))) <
                  kappa (recDelta b s (delta (recDelta b' s' n'))) := by
          simp [kappa]
          admit
        exact drop_left hk

    | eqW a b' =>
        have hk : kappa (app s (recDelta b s (eqW a b'))) < kappa (recDelta b s (delta (eqW a b'))) := by
          simp [kappa]
          admit
        exact drop_left hk
```

### What you should take away
- The above is the **right “shape”** of the proof: case split on `n`, use `drop_left` in each branch.
- But **the `n = delta n'` branch is genuinely blocked** with the current measure, for the reason written in the comments: `kappa` ties and `mu` increases.

If your goal is a *skeleton only*, the above pinpoints where the real work is: you’ll need to **refine the measure** (or the notion of step/context) so that this “delta-delta” branch becomes strictly decreasing in the first component again.

## `R_eq_refl` : `eqW a a → void`

Same pattern as `R_int_delta`: split on `kappa a`.

```lean
| _, _, Step.R_eq_refl a => by
    -- Goal: LexOrder (measure void) (measure (eqW a a)).
    cases h : kappa a with
    | zero =>
        have hk : kappa void = kappa (eqW a a) := by
          simp [kappa, h]
        have hmu : mu void < mu (eqW a a) := by
          simp [mu]
          omega
        exact drop_right hmu hk
    | succ k =>
        have hk : kappa void < kappa (eqW a a) := by
          simpa [kappa, h] using (Nat.succ_pos k)
        exact drop_left hk
```

## `R_eq_diff` : `eqW a b → integrate (merge a b)`

Here `kappa` ties *definitionally* and `mu` drops by exactly `1`, so it’s a clean `drop_right`.

```lean
| _, _, Step.R_eq_diff a b => by
    have hmu : mu (integrate (merge a b)) < mu (eqW a b) := by
      simp [mu]
      omega
    have hk : kappa (integrate (merge a b)) = kappa (eqW a b) := by
      simp [kappa]
    exact drop_right hmu hk
```
