## Assessment

### If “the system” is literally `Step` as defined (root-only)
Yes — strong normalization is straightforward and can be established from the presented calculus alone.

Reason: every `Step` rule either
- strictly simplifies a constructor shape (`integrate (delta _) → void`, merge simplifications, `eqW _ _ → ...`), or
- performs a bounded primitive-recursion-style unfold on the **leading** `delta` of the third argument (`recΔ b s (delta n) → app s (recΔ b s n)`), which can only fire as many times as there are leading `delta` constructors available at that position (and no rule ever *creates* a new outermost `delta`).

Even though `R_rec_succ` duplicates `s` (it appears both as the `app` head and as an argument to the recursive call), that duplication happens a **finite** number of times because it’s tied to consuming a finite `delta` spine on the recursor’s third argument.

### If “the system” means the usual *full* term-rewriting closure (rewrite in any context)
That closure is **not present in this file**, so you can’t claim you proved SN of *that* system “from the presented calculus alone” unless you additionally **define** the contextual closure relation.

However: if you *do* take the standard TRS reading (compatible/contextual closure) as the intended notion, then **SN still looks provable without adding new rewrite rules/guards/side conditions**, using a standard termination order (e.g. a lexicographic path order / recursive path order) that orients every rule. The only “extra” ingredient would be meta-theory about that order being well-founded—i.e. proof infrastructure, not object-level axioms.

## Short roadmap (stays within the boundary)

I’ll phrase this in a way that works for the *standard TRS interpretation*; if you stay root-only it becomes even simpler.

1. **Define** the actual reduction relation you care about:
   - either root-only: just `Step`,
   - or full rewriting: the smallest relation closed under contexts that contains `Step`.
   (This is a definition, not a new rewrite rule.)

2. **Pick a well-founded simplification order** on `Trace` terms (standard choice: **LPO/RPO**) with a precedence such as:
   - `recΔ`  > `app`
   - `eqW`   > `integrate` > `merge`
   - `integrate` > `void` (constants lowest)
   (The rest can be arranged arbitrarily as long as it doesn’t break the checks below.)

3. **Check each rewrite rule decreases in the order**:
   - `merge ... → ...` rules: RHS is a **proper subterm** of LHS, so they decrease by the subterm property.
   - `recΔ b s void → b`: RHS `b` is a subterm of LHS.
   - `recΔ b s (delta n) → app s (recΔ b s n)`:
     - With `recΔ > app`, it suffices to show `recΔ(b,s,delta n)` is greater than each argument of the RHS.
     - It’s > `s` because `s` is a subterm of LHS.
     - It’s > `recΔ(b,s,n)` because they have the same top symbol and (with lex status) the third argument decreases: `delta n > n` (again by subterm property).
   - `integrate (delta t) → void`: ensured by precedence `integrate > void` (or more generally, any compound term > constant).
   - `eqW a a → void`: ensured by `eqW > void`.
   - `eqW a b → integrate (merge a b)`:
     - With `eqW > integrate`, it suffices that `eqW(a,b) > merge(a,b)`.
     - With `eqW > merge`, it suffices that `eqW(a,b) > a` and `> b`, which holds since `a,b` are subterms.

4. Conclude **no infinite descending chain**, hence **no infinite rewrite sequence** (SN).

## Why this stays “within the calculus”
- You’re not adding any new rewrite rules, side conditions, or axioms *about* `Trace`.
- You’re proving a meta-property of the presented rewrite rules using a standard well-founded order argument.

The only potential “boundary dispute” is whether you regard *introducing the contextual closure relation* as “importing structural axioms.” If you do, then the strongest statement you can make **from this file alone** is SN of the *root-step* relation `Step` (which, again, should be even easier than the LPO route).
