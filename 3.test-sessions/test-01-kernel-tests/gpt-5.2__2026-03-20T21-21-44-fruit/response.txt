## Why this looks strongly normalizing

A quick “cycle sniff test”:

- Most rules are **collapsing** to a subterm (`peach … → t`, `banana … plum → b`) or to a constant (`… → plum`), which is termination-friendly.
- The only rule that *looks* size-increasing is:

\[
\texttt{banana b s (grape n)} \to \texttt{pear s (banana b s n)}
\]

but it strictly “peels off” one `grape` from the 3rd argument of `banana`. Even though it introduces a `pear`, it does not create a mechanism to re-introduce that `grape` layer, so it still behaves like descent.

## Short proof roadmap (stays within the given boundary)

The cleanest standard method here is **Recursive Path Ordering (RPO)** (or any equivalent simplification order), which is:

- well-founded,
- closed under contexts/subterms (i.e., works for the usual “rewrite anywhere in a term” notion),
- requires no extra rewrite rules or axioms—just a precedence on function symbols.

### 1) Choose a precedence on constructors

Pick a precedence with `banana` above `pear`, and `cherry` above `mango` and `peach`, etc. For example:

\[
\texttt{banana} \succ \texttt{pear} \succ \texttt{cherry} \succ \texttt{mango} \succ \texttt{peach} \succ \texttt{grape} \succ \texttt{plum}
\]

(Exact middle ordering can vary; the key constraints are `banana ≻ pear` and `cherry ≻ mango` and `cherry ≻ peach`.)

Use lexicographic status for arguments (common default).

### 2) Check each rule decreases under RPO

Most are immediate because the RHS is a proper subterm or a smaller-root term:

- `peach plum t → t`, `peach t plum → t`, `peach t t → t` are strict by the **subterm property**.
- `banana b s plum → b` strict by subterm property.
- `mango (grape t) → plum` strict since `mango ≻ plum`.
- `cherry a a → plum` strict since `cherry ≻ plum`.
- Conditional rule `cherry a b → mango (peach a b)` (given `a ≠ b`): strict since `cherry ≻ mango`, and also `cherry a b` is above `peach a b` because `cherry ≻ peach` and its arguments dominate `a` and `b`.

The only “interesting” one is the `banana/grape` rule:

\[
\texttt{banana b s (grape n)} \to \texttt{pear s (banana b s n)}
\]

This is oriented because:

- root decreases: `banana ≻ pear`, and
- you can show `banana b s (grape n) ≻ s` (subterm), and
- crucially `banana b s (grape n) ≻ banana b s n` because with lex comparison the first two args match and you compare `grape n ≻ n` (again by subterm property: a term is bigger than its proper subterms).

So the RHS is strictly smaller.

### 3) Conclude strong normalization

Since all rules strictly decrease under a well-founded reduction order that is stable under contexts, there cannot be an infinite rewrite sequence. Hence the system is strongly normalizing (terminating).
