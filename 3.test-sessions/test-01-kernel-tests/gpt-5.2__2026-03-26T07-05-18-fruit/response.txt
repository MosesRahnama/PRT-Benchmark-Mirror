## Why it terminates (core idea)

You can orient all rules by a **well‚Äëfounded lexicographic measure** on terms defined purely from the syntax.

A robust choice (works even if you later take contextual closure) is the triple
\[
\mu(t) := \big(\#\text{cherry}(t),\ \#\text{grape}(t),\ \text{size}(t)\big)
\]
ordered lexicographically on $\mathbb{N}^3$.

Define these structurally (primitive recursion on `Trace`):

- `#cherry(t)`: number of `cherry` nodes occurring anywhere in `t`
- `#grape(t)`: number of `grape` nodes occurring anywhere in `t`
- `size(t)`: total number of constructors in `t` (each node counts as 1 + sizes of children)

### Check each rewrite rule decreases \(\mu\)

- `R_cherry_refl : cherry a a ‚Üí plum`  
  Decreases `#cherry` by 1 (so lex decreases immediately).

- `R_cherry_diff : cherry a b ‚Üí mango (peach a b)`  
  Also decreases `#cherry` by 1 (even though `size` goes *up* by 1, lex order doesn‚Äôt care once the first component drops).

- `R_apple_orange : banana b s (grape n) ‚Üí pear s (banana b s n)`  
  Decreases `#grape` by 1 (the outer `grape` disappears). `size` increases, but lex order decreases at the second component.

- `R_mango_grape : mango (grape t) ‚Üí plum`  
  Decreases `#grape` by 1 (and size drops too).

- `R_peach_plum_left/right/cancel` and `R_banana_zero`  
  Preserve `#cherry` and `#grape`, but **strictly decrease `size`** (they drop at least one constructor node and don‚Äôt introduce any new ones).

So every one-step rewrite strictly decreases \(\mu\) in lex order. Since lex on \(\mathbb{N}^3\) is well-founded, there cannot be an infinite reduction sequence.

## Short proof roadmap (stays within the given system)

- Define functions on `Trace`:
  - `countCherry : Trace ‚Üí Nat`
  - `countGrape : Trace ‚Üí Nat`

- Let `mu t := (countCherry t, countGrape t, size t)` and use lex order on triples of `Nat`.
- Prove lemma: for any `h : Step t u`, we have `mu u <lex mu t` by cases on `h` (one case per constructor of `Step`).
- Conclude **strong normalization**: any reduction sequence yields a strictly descending chain in a well-founded order, impossible.

That argument uses only:
- structural recursion on the given inductive syntax
- case analysis on the given inductive rewrite rules
- well-foundedness of lex order on `Nat` tuples (meta-theoretic, not an extra object-level axiom about `Trace`)
