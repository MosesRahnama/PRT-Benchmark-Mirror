## Raw assessment

**Yes — strong normalization of the relation `Step` as defined in this file can be established from the presented calculus alone**, with no extra object-level axioms and without adding any new rewrite rules/guards/side conditions.

The key (and it matters a lot) is that `Step : Trace → Trace → Prop` here is **not** the usual “rewrite anywhere inside a term” relation: there are **no congruence/context rules** like
- if `Step t t'` then `Step (grape t) (grape t')`, etc.

So reductions happen only when the *entire term* matches one of the listed left-hand sides.

Under that exact calculus, there is no mechanism for an infinite rewrite sequence:
- Many rules strictly “project” to an argument (a proper subterm): the `peach` rules and `banana b s plum → b`.
- The two rules that *increase* size (`banana … (grape n) → pear … (banana … n)` and `cherry a b → mango (peach a b)` when `a ≠ b`) both land in shapes whose **root constructor has no outgoing `Step` rules** (`pear …` has none; `mango (peach …)` can’t match `mango (grape …)`), hence they terminate immediately.

So every `Step`-chain is finite.

## In-boundary proof roadmap (Lean-friendly)

A standard, assumption-free way is to prove accessibility:

1. **Define strong normalization** as accessibility:
   - `SN (t : Trace) : Prop := Acc Step t`
   - Goal: `theorem strong_normalization : ∀ t, Acc Step t`

2. **Prove “no outgoing step” lemmas** by contradiction + `cases` on a hypothetical `Step` proof:
   - `noStep_plum : ¬ ∃ u, Step plum u`
   - `noStep_grape : ¬ ∃ u, Step (grape t) u`
   - `noStep_pear : ¬ ∃ u, Step (pear a b) u`
   - `noStep_mango_peach : ¬ ∃ u, Step (mango (peach a b)) u`
     (because the only `mango` rule is `mango (grape t) → plum`, and `peach _ _` can’t unify with `grape _`)

3. **Main proof by structural recursion on `t : Trace`**:
   - For each constructor case, build `Acc.intro` and show every immediate successor is `Acc`.
   - When `Step` returns a **subterm** (e.g. `peach plum t → t`, `banana b s plum → b`, `peach t t → t`), discharge via the recursion hypothesis on that argument.
   - When `Step` returns one of the **stuck shapes** (`pear …` or `mango (peach …)`), discharge using the “no outgoing step” lemmas (accessibility is trivial if there are no successors).

Concretely, the only mildly “non-subterm” successors are:
- `R_apple_orange`: `banana b s (grape n) → pear s (banana b s n)`; prove `Acc` because `pear …` has no steps at the root.
- `R_cherry_diff`: `cherry a b → mango (peach a b)`; prove `Acc` because `mango (peach …)` has no steps at the root.

That’s it; no decidable equality, no ordering on terms, no imported rewriting theory required.
