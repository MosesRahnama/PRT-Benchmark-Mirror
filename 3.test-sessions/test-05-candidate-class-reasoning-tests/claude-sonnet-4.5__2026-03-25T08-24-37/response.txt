## Assessment of Termination Measures

For a measure to prove termination, it must **strictly decrease** for every rewrite step. Let me check each rule:

### Rules that all three measures handle correctly:

**R_int_delta**: `integrate (delta t)` → `void`
- All measures: (constant + measure(t)) → 0 ✓ Always decreases

**R_merge_void_left/right**: `merge void t` → `t` 
- All measures: (constant + measure(t)) → measure(t) ✓ Decreases by the constant

**R_merge_cancel**: `merge t t` → `t`
- All measures: (constant + 2×measure(t)) → measure(t) ✓ Decreases significantly

**R_rec_zero**: `recDelta b s void` → `b`
- All measures: (constant + measure(b) + measure(s)) → measure(b) ✓ Decreases by constant + measure(s)

**R_eq_refl**: `eqW a a` → `void`
- All measures: (constant + 2×measure(a)) → 0 ✓ Decreases

**R_eq_diff**: `eqW a b` → `integrate (merge a b)`

- mu3: (4 + μ₃(a) + μ₃(b)) → (3 + μ₃(a) + μ₃(b)) ✓ Decreases by 1

### **Critical failure: R_rec_succ**

`recDelta b s (delta n)` → `app s (recDelta b s n)`

## Conclusion

**All three candidates fail.** None of them are valid termination measures for this rewrite system. The recursive descent pattern in `R_rec_succ` causes the measure to increase because `app` duplicates its first argument `s` while `recDelta` only counts it once. The measure "grows" an extra copy of `μ(s)` on each recursive step, making termination unprovable with these simple size-based measures.
