**All three candidates fail on `R_rec_succ`.** The fundamental issue is that this rule duplicates the subterm `s`:

```

```

The RHS contains `s` **twice** (once directly, once inside the recursive `recDelta`), while the LHS contains it once. No simple additive/weighted node-counting measure of the form $\mu(t) = c + \sum \mu(\text{children})$ can absorb this duplication вЂ” it would require $c_{\text{recDelta}} > c_{\text{app}} + s$ for all `s`, which is impossible when `s` is unbounded.
