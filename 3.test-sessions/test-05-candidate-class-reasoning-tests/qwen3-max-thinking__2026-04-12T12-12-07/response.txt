### R_rec_succ: `recDelta b s (delta n) → app s (recDelta b s n)`

This is the critical rule. The rhs is `app s (recDelta b s n)`.

**μ1:**
- lhs: $1 + b + s + (1 + n) = 2 + b + s + n$
- rhs: $1 + s + (1 + b + s + n) = 2 + b + 2s + n$
- Δ = lhs − rhs = $-s$

This is **negative when $s > 0$**. μ1 **fails** on `R_rec_succ`.

**μ2:**
- lhs: $5 + b + s + (1 + n) = 6 + b + s + n$
- rhs: $1 + s + (5 + b + s + n) = 6 + b + 2s + n$
- Δ = $-s$

Same failure. μ2 **fails** on `R_rec_succ`.

**μ3:**
- lhs: $1 + b + s + (1 + n) = 2 + b + s + n$
- rhs: $2 + s + (1 + b + s + n) = 3 + b + 2s + n$
- Δ = $-1 - s$

Strictly negative. μ3 **fails** on `R_rec_succ`.

## Verdict

All three candidates **fail** on `R_rec_succ`.

The structural reason is shared: the rhs copies `s` into both the head position of `app` and the recursive call `recDelta b s n`, so `s` appears **twice** on the right-hand side but only once on the left. No purely additive, subterm-linear measure can survive this unless the overhead assigned to `recDelta` strictly exceeds the overhead assigned to `app` plus an extra copy of `s`. Concretely, what is needed is:

$$\text{overhead}(\mathtt{recDelta}) > \text{overhead}(\mathtt{app}) + \mu(s)$$

But since $\mu(s)$ is unbounded (it depends on the actual subtree), no **fixed** constant overhead can satisfy this inequality. A valid termination argument therefore cannot be a simple structural size; it must use the `delta`-depth of the third argument (the counter) as a lexicographic component, decreasing on `delta n → n`, with `b` and `s` held constant in the recursive call.
