# Assessment of Termination Measure Candidates

I've analyzed the three proposed termination measures (mu1, mu2, and mu3) for the first-order rewrite calculus defined in the Lean 4 file. A valid termination measure must strictly decrease with each rewrite step.

## Analysis of Each Rule

### R_int_delta: `integrate (delta t) → void`
- **mu1**: LHS = 1 + (1 + mu1 t) = 2 + mu1 t, RHS = 0
  - Decreases only if mu1 t ≥ 0 (which is true), but 2 + mu1 t > 0 always
  - **Valid**: Yes, always decreases

- **mu2**: LHS = 2 + (1 + mu2 t) = 3 + mu2 t, RHS = 0
  - **Valid**: Yes, always decreases

- **mu3**: LHS = 2 + (1 + mu3 t) = 3 + mu3 t, RHS = 0
  - **Valid**: Yes, always decreases

### R_merge_void_left/right: `merge void t → t` and `merge t void → t`
- **mu1**: LHS = 1 + 0 + mu1 t = 1 + mu1 t, RHS = mu1 t
  - **Valid**: Yes, decreases by 1

- **mu2**: LHS = 2 + 0 + mu2 t = 2 + mu2 t, RHS = mu2 t
  - **Valid**: Yes, decreases by 2

- **mu3**: LHS = 1 + 0 + mu3 t = 1 + mu3 t, RHS = mu3 t
  - **Valid**: Yes, decreases by 1

### R_merge_cancel: `merge t t → t`
- **mu1**: LHS = 1 + mu1 t + mu1 t = 1 + 2*mu1 t, RHS = mu1 t
  - **Valid**: Yes, decreases by 1 + mu1 t (≥ 1)

- **mu2**: LHS = 2 + mu2 t + mu2 t = 2 + 2*mu2 t, RHS = mu2 t
  - **Valid**: Yes, decreases by 2 + mu2 t (≥ 2)

- **mu3**: LHS = 1 + mu3 t + mu3 t = 1 + 2*mu3 t, RHS = mu3 t
  - **Valid**: Yes, decreases by 1 + mu3 t (≥ 1)

### R_rec_zero: `recDelta b s void → b`
- **mu1**: LHS = 1 + mu1 b + mu1 s + 0 = 1 + mu1 b + mu1 s, RHS = mu1 b
  - **Valid**: Yes, decreases by 1 + mu1 s (≥ 1)

- **mu2**: LHS = 5 + mu2 b + mu2 s + 0 = 5 + mu2 b + mu2 s, RHS = mu2 b
  - **Valid**: Yes, decreases by 5 + mu2 s (≥ 5)

- **mu3**: LHS = 1 + mu3 b + mu3 s + 0 = 1 + mu3 b + mu3 s, RHS = mu3 b
  - **Valid**: Yes, decreases by 1 + mu3 s (≥ 1)

### R_rec_succ: `recDelta b s (delta n) → app s (recDelta b s n)`
- **mu1**: 
  - LHS = 1 + mu1 b + mu1 s + (1 + mu1 n) = 2 + mu1 b + mu1 s + mu1 n
  - RHS = 1 + mu1 s + (1 + mu1 b + mu1 s + mu1 n) = 2 + mu1 b + 2*mu1 s + mu1 n
  - **Valid**: No! RHS = LHS + mu1 s ≥ LHS (increases when mu1 s > 0)

- **mu2**:
  - LHS = 5 + mu2 b + mu2 s + (1 + mu2 n) = 6 + mu2 b + mu2 s + mu2 n
  - RHS = 1 + mu2 s + (5 + mu2 b + mu2 s + mu2 n) = 6 + mu2 b + 2*mu2 s + mu2 n
  - **Valid**: No! RHS = LHS + mu2 s ≥ LHS (increases when mu2 s > 0)

- **mu3**:
  - LHS = 1 + mu3 b + mu3 s + (1 + mu3 n) = 2 + mu3 b + mu3 s + mu3 n
  - RHS = 2 + mu3 s + (1 + mu3 b + mu3 s + mu3 n) = 3 + mu3 b + 2*mu3 s + mu3 n
  - **Valid**: No! RHS = LHS + 1 + mu3 s > LHS (always increases)

### R_eq_refl: `eqW a a → void`
- **mu1**: LHS = 3 + mu1 a + mu1 a = 3 + 2*mu1 a, RHS = 0
  - **Valid**: Yes, always decreases

- **mu2**: LHS = 5 + mu2 a + mu2 a = 5 + 2*mu2 a, RHS = 0
  - **Valid**: Yes, always decreases

- **mu3**: LHS = 4 + mu3 a + mu3 a = 4 + 2*mu3 a, RHS = 0
  - **Valid**: Yes, always decreases

### R_eq_diff: `eqW a b → integrate (merge a b)`
- **mu1**: 
  - LHS = 3 + mu1 a + mu1 b
  - RHS = 1 + (1 + mu1 a + mu1 b) = 2 + mu1 a + mu1 b
  - **Valid**: Yes, decreases by 1

- **mu2**:
  - LHS = 5 + mu2 a + mu2 b
  - RHS = 2 + (2 + mu2 a + mu2 b) = 4 + mu2 a + mu2 b
  - **Valid**: Yes, decreases by 1

- **mu3**:
  - LHS = 4 + mu3 a + mu3 b
  - RHS = 2 + (1 + mu3 a + mu3 b) = 3 + mu3 a + mu3 b
  - **Valid**: Yes, decreases by 1

## Conclusion

All three measures fail to be valid termination measures because they all fail on the `R_rec_succ` rule. In each case, the measure increases rather than decreases when applying this rule.

The fundamental issue is in how the measures handle the `recDelta` and `app` constructors. When `recDelta b s (delta n)` rewrites to `app s (recDelta b s n)`, the right-hand side contains both `s` and a recursive call to `recDelta`, which causes the measure to increase.

To fix this, a valid termination measure would need to assign a lower weight to the `app` constructor or a higher weight to the nested structure in `recDelta`, or use a more sophisticated measure like a lexicographic or multiset ordering rather than a simple natural number measure.
