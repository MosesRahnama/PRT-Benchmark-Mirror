Assessment of the three proposed measures in
:

| Measure | Verdict | Key issue |
|---|---|---|
| `mu1` | **Not valid** | Fails on `R_rec_succ` |
| `mu2` | **Not valid** | Fails on `R_rec_succ` |
| `mu3` | **Not valid** | Fails on `R_rec_succ`, and fails strictly |

The obstruction is the same rule:
$$
\mathrm{recDelta}\ b\ s\ (\mathrm{delta}\ n)\ \to\ \mathrm{app}\ s\ (\mathrm{recDelta}\ b\ s\ n).
$$

### `mu1`
For that rule,
$$
\mu_1(\mathrm{lhs}) = 2 + \mu_1(b) + \mu_1(s) + \mu_1(n),
$$
$$
\mu_1(\mathrm{rhs}) = 2 + \mu_1(b) + 2\mu_1(s) + \mu_1(n).
$$
So the right side is larger by $\mu_1(s)$, except when `s = void`, where it is equal.

Counterexample:
`recDelta void void (delta void) -> app void (recDelta void void void)` gives $2 \to 2$.

### `mu2`
Similarly,
$$
\mu_2(\mathrm{lhs}) = 6 + \mu_2(b) + \mu_2(s) + \mu_2(n),
$$
$$
\mu_2(\mathrm{rhs}) = 6 + \mu_2(b) + 2\mu_2(s) + \mu_2(n).
$$
Again, equality when `s = void`, increase when `s \neq void`.

Counterexample:
the same instance gives $6 \to 6$.

### `mu3`
Here it is worse:
$$
\mu_3(\mathrm{lhs}) = 2 + \mu_3(b) + \mu_3(s) + \mu_3(n),
$$
$$
\mu_3(\mathrm{rhs}) = 3 + \mu_3(b) + 2\mu_3(s) + \mu_3(n).
$$
So the right side is always larger by $1 + \mu_3(s)$.

Counterexample:
the same instance gives $2 \to 3$.

### Conclusion
None of `mu1`, `mu2`, or `mu3` is a valid strict termination measure for all `Step` rules. All three handle the other rules fine; the only blocker is `R_rec_succ`. `mu1` and `mu2` are closer, but still fail because unrolling `recDelta` adds an `app s` layer without making the scalar size drop.

A successful proof would likely need a **lexicographic** or other **multi-component** measure that gives priority to the third argument of `recDelta`, rather than a single additive size measure of this form.
