**Assessment of the three termination‑measure candidates**  

The rewrite calculus is defined by the inductive type `Trace` and the relation `Step`.  
A termination measure `μ : Trace → Nat` must be *well‑founded* (always non‑negative) and must **strictly decrease** for every rewrite step `Step t₁ t₂`.

Below we examine each of the three measures (`mu1`, `mu2`, `mu3`) against the seven rewrite rules.

| Rule | Left‑hand side (LHS) | Right‑hand side (RHS) | What the rule does | Behaviour of the measure |
|------|----------------------|-----------------------|--------------------|--------------------------|
| **R_int_delta** | `integrate (delta t)` | `void` | Eliminates a `delta` wrapped in `integrate`. | All three measures give a positive value to the LHS (≥ 2) and 0 to `void`; they decrease. |
| **R_merge_void_left / right** | `merge void t` / `merge t void` | `t` | Removes a `void` argument from a `merge`. | `μ(LHS) = c + μ t` (c = 1 for `mu1`, 2 for `mu2`, 1 for `mu3`); RHS = `μ t`. Decrease by the constant `c`. |
| **R_merge_cancel** | `merge t t` | `t` | Cancels a duplicate argument. | `μ(LHS) = c + 2·μ t` > `μ t`. Decrease. |
| **R_rec_zero** | `recDelta b s void` | `b` | Drops the `s` argument when the recursion index is `void`. | `μ(LHS) = c₁ + μ b + μ s` (c₁ = 1, 5, 1 respectively) > `μ b`. Decrease. |
| **R_eq_refl** | `eqW a a` | `void` | Trivial equality. | All measures give a positive value to the LHS (≥ 3) and 0 to `void`. Decrease. |
| **R_eq_diff** | `eqW a b` | `integrate (merge a b)` | Rewrites an inequality into a composition. | For each measure `μ(LHS) = c₂ + μ a + μ b` (c₂ = 3, 5, 4) and `μ(RHS) = c₃ + μ a + μ b` (c₃ = 1, 2, 2). Since `c₃ < c₂`, the measure **decreases**. |
| **R_rec_succ** | `recDelta b s (delta n)` | `app s (recDelta b s n)` | Performs one recursive step, decreasing the index from `delta n` to `n` but re‑introducing `s`. | **All three measures increase** (unless `s = void`).  <br>*Computation* (using `mu1` as example): <br>`μ(LHS) = 2 + μ b + μ s + μ n` <br>`μ(RHS) = 2 + μ b + 2·μ s + μ n` <br>→ RHS = LHS + μ s. <br>Identical calculations for `mu2` and `mu3` give RHS = LHS + μ s (or RHS = LHS + (1+μ s) for `mu3`). Thus the measure **fails to decrease** whenever `s` is non‑void. |

### Summary of the failures

* The only rule that breaks the decreasing‑measure property for **all three candidates** is **`R_rec_succ`**.  
* The increase is proportional to the size of the second argument `s`. Since `s` can be an arbitrary (non‑void) trace, the measures are not guaranteed to descend on this rule.  
* Consequently, none of the three candidates constitutes a valid global termination measure for the rewrite system.

### Comparative remarks

| Measure | Strengths | Weaknesses |
|---------|-----------|------------|
| **`mu1`** | Simple uniform weight (1) for most constructors; gives a modest penalty (3) to `eqW`. | Still increases on `R_rec_succ`; does not distinguish the recursive index (`delta n`) from the carried argument `s`. |
| **`mu2`** | Gives higher weight to `integrate`, `merge`, and `recDelta`, which helps for `R_eq_diff` and `R_rec_zero`. | The extra weight does not offset the extra copy of `s` introduced by `R_rec_succ`; the measure still grows. |
| **`mu3`** | Tries to lower the cost of `merge` and `recDelta` (weight 1) while raising `eqW` (weight 4). | The reduction in weight for `recDelta` is insufficient; `R_rec_succ` still adds at least `1 + μ s`. |

### Conclusion

All three candidates (`mu1`, `mu2`, `mu3`) **fail** to be proper termination measures for the given rewrite calculus because they do **not** strictly decrease on the `R_rec_succ` rule when the second argument `s` is non‑void. A correct measure would need to assign a *larger* weight to the recursive index (`delta n`) than to the carried argument `s`, ensuring that the overall value drops after each `R_rec_succ` step. None of the provided candidates satisfies this requirement.
