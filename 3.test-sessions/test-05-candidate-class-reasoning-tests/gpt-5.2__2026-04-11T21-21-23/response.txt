All three proposed measures fail as *global* termination measures for `Step`, because the recursion-unfolding rule `R_rec_succ` duplicates the parameter `s` (it appears once in the outer `app` and again inside the remaining `recDelta b s n`). Any measure that “adds up sizes” of all subterms will typically stay the same or increase there.

**mu1**
- Every rule except `R_rec_succ` strictly decreases `mu1`.
- For `R_rec_succ`:
  - LHS: `mu1 (recDelta b s (delta n)) = 2 + mu1 b + mu1 s + mu1 n`
  - RHS: `mu1 (app s (recDelta b s n)) = 2 + mu1 b + 2·mu1 s + mu1 n`
  - So `mu1(RHS) - mu1(LHS) = mu1 s` (nonnegative), meaning it does **not** strictly decrease; it **increases** whenever `s ≠ void`.
- Concrete counterexample (strict increase): take `b = void`, `s = delta void`, `n = void`:
  - `mu1(recDelta void (delta void) (delta void)) = 3`
  - `mu1(app (delta void) (recDelta void (delta void) void)) = 4`

**mu2**
- Same structural issue on `R_rec_succ`:
  - LHS: `mu2 (recDelta b s (delta n)) = 6 + mu2 b + mu2 s + mu2 n`
  - RHS: `mu2 (app s (recDelta b s n)) = 6 + mu2 b + 2·mu2 s + mu2 n`
  - Hence `mu2(RHS) - mu2(LHS) = mu2 s` (nonnegative), so not strictly decreasing; increases if `s ≠ void`.
- Concrete counterexample: `b = void`, `s = delta void`, `n = void`:
  - `mu2(LHS) = 7`, `mu2(RHS) = 8`

**mu3**
- `R_rec_succ` is even worse: it **always strictly increases**, even when `s = void`:
  - LHS: `mu3 (recDelta b s (delta n)) = 2 + mu3 b + mu3 s + mu3 n`
  - RHS: `mu3 (app s (recDelta b s n)) = 3 + mu3 b + 2·mu3 s + mu3 n`
  - So `mu3(RHS) - mu3(LHS) = 1 + mu3 s ≥ 1`.
- Example with `s = void`: `mu3(RHS) = mu3(LHS) + 1`.

**Bottom line**
- None of `mu1`, `mu2`, `mu3` can serve as a single natural-number ranking function that strictly decreases on every `Step`.
- The only problematic rule is `R_rec_succ`; the other seven rules decrease all three measures.
