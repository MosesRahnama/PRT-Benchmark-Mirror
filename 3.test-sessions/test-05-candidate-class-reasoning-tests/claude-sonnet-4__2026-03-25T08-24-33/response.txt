**R_int_delta: `integrate (delta t) → void`**

### Critical failure point:

**R_rec_succ: `recDelta b s (delta n) → app s (recDelta b s n)`**

Let me calculate each measure:

**μ₁ analysis:**
- LHS: `μ₁(recDelta b s (delta n)) = 1 + μ₁(b) + μ₁(s) + (1 + μ₁(n)) = 2 + μ₁(b) + μ₁(s) + μ₁(n)`
- RHS: `μ₁(app s (recDelta b s n)) = 1 + μ₁(s) + (1 + μ₁(b) + μ₁(s) + μ₁(n)) = 2 + μ₁(b) + 2μ₁(s) + μ₁(n)`
- Required: `2 + μ₁(b) + μ₁(s) + μ₁(n) > 2 + μ₁(b) + 2μ₁(s) + μ₁(n)`
- Simplifies to: `μ₁(s) > 2μ₁(s)` — **Never true for μ₁(s) ≥ 0** ✗

**μ₂ analysis:**
- LHS: `6 + μ₂(b) + μ₂(s) + μ₂(n)`
- RHS: `6 + μ₂(b) + 2μ₂(s) + μ₂(n)`
- Simplifies to: `μ₂(s) > 2μ₂(s)` — **Never true** ✗

**μ₃ analysis:**
- LHS: `2 + μ₃(b) + μ₃(s) + μ₃(n)`
- RHS: `3 + μ₃(b) + 2μ₃(s) + μ₃(n)`
- Simplifies to: `2 + μ₃(s) > 3 + 2μ₃(s)` → `-1 > μ₃(s)` — **Never true** ✗

## Conclusion

**All three measures fail** as termination measures for this rewrite system. They all fail on the same rule: `R_rec_succ`. This rule represents the recursive unfolding of `recDelta` when applied to a successor (`delta n`), and the structural issue is that the right-hand side duplicates the `s` argument in two places (`app s (recDelta b s n)`), causing the measure to increase rather than decrease.

The fundamental problem is that none of these simple additive measures can handle the duplication inherent in recursive unfolding. A successful termination measure for this system would need to account for the fact that `R_rec_succ` is meant to make progress by reducing the third argument from `delta n` to `n`, even though it temporarily increases the overall term size.
