The decisive obstruction is the recursive step KO7-LLM-Benchmark/test-files/Test-05-Candidate-Class-Reasoning.lean:
`recDelta b s (delta n) -> app s (recDelta b s n)`

This rule keeps the old `s` inside `recDelta` and also adds a new outer occurrence of `s` as an argument to `app`, so the right-hand side contains one extra copy of `s`. Since all three `mu` functions are additive and assign nonnegative weight to subterms, that extra copy is exactly where they break.

For the three candidates:

`mu1(lhs) = 2 + mu1 b + mu1 s + mu1 n`
`mu1(rhs) = 2 + mu1 b + 2 * mu1 s + mu1 n`
So `mu1(rhs) - mu1(lhs) = mu1 s`.
If `s = void`, the measure is equal, not strictly smaller.
If `s` is nontrivial, the measure increases.

2. `mu2` is not valid.
`mu2(lhs) = 6 + mu2 b + mu2 s + mu2 n`
`mu2(rhs) = 6 + mu2 b + 2 * mu2 s + mu2 n`
Again `mu2(rhs) - mu2(lhs) = mu2 s`.
So it has the same failure mode as `mu1`: equality when `s = void`, increase otherwise.

`mu3(lhs) = 2 + mu3 b + mu3 s + mu3 n`
