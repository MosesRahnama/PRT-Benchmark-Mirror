Based on just these rules, I would not say strong normalization is established. The system has no variables binding and only two rules, but termination is not immediate: the second rule rewrites `F(x, y, S(n))` to `G(F(x, y, n))`, which introduces an extra `G` wrapper around a smaller `F` argument. That looks like a primitive recursion on `n`, but the system lacks any rule that reduces or eliminates `G`, and there is no rule that relates `G(F(...))` back to an `F`-redex. So the rewrite relation can get stuck in a `G(...)` context, and the only potential for infinite reduction would be if `G` participates in further rules (it does not here). 

From these rules alone, every reduction starting at an `F`-term strictly decreases the `S`-depth of its third argument until it reaches `Z`, and then stops. So I see no infinite rewrite sequence, but that conclusion relies on interpreting `n` as a finite term built from `Z` and `S`, which is implicit in first-order rewriting. If you require a formal termination proof solely from the rules without any structural assumptions about term formation or constructor well-foundedness, the system does not encode that by itself.
